# Connectors — 调查工具连接器

cc-investigation 的 MCP 连接器生态。每个连接器为调查技能提供外部数据或工具能力。

## 连接器分类

cc-investigation 的 MCP 连接器按部署方式分为三类：

| 类别 | 说明 | 配置方式 |
|------|------|---------|
| **🟢 自动部署** | 插件自带，无需任何配置即可使用 | 通过项目 `.mcp.json` 自动加载 |
| **🔵 内置能力** | Claude Code 原生工具，无需 MCP 配置 | 直接使用（WebSearch、WebFetch 等） |
| **🟡 用户配置** | 需要用户手动注册，部分需要 API Key | 添加到 `.mcp.json` 或 `~/.claude.json` |
| **⚪ 占位** | 有需求但暂无成熟社区包 | 待社区生态成熟后接入 |

## 配置指南

### 自动部署（无需操作）

`investigation-pdf` 已在 `.mcp.json` 中注册，每次启动 Claude Code 时自动加载。

### 用户配置（按需添加）

如需启用用户配置的连接器，在项目 `.mcp.json` 或 `~/.claude.json` 中添加：

```json
{
  "mcpServers": {
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@brave/brave-search-mcp-server"],
      "env": {
        "BRAVE_API_KEY": "your-free-api-key"
      }
    },
    "baidu-map": {
      "command": "npx",
      "args": ["-y", "@baidumap/mcp-server-baidu-map"],
      "env": {
        "BAIDU_MAP_API_KEY": "your-api-key"
      }
    }
  }
}
```

配置完成后，运行 `/cc-investigation:cold-start-interview --check-integrations` 验证集成状态。

## 添加新 Connector

如果你构建或运营一个调查数据源（企业信息查询、裁判文书、银行流水分析等），欢迎你的 MCP connector 加入。

### 合格标准

- **远程 MCP over HTTPS**，OAuth 或 API Key 认证
- **以读为主的工具**（查询、搜索、获取）
- **结果带来源标注** — 返回来源、检索日期、可引用的标识符
- **无指令伪装成数据** — 返回内容中的元数据需清晰标记
- **优雅降级** — 连接不可用时返回明确错误，而非超时

### 提交流程

1. 发布你的 MCP 服务器并文档化工具和认证方式
2. 提交 PR 将你的服务器加入 `mcp-configs/mcp-servers.json`
3. 标注服务于哪些技能
4. 通过注入安全检查和检索质量评估后合并

## ⚠️ 隐私、安全与合规说明

### 核心原则
- **连接器配置中的 API Key 等敏感信息**不**存储在配置模板中**。用户在自己的 `~/.claude.json` 或项目 settings.json 中配置凭据。
- **涉及敏感数据的连接器（案件数据库、证据文件系统）务必限定访问范围**，遵循最小权限原则。
- **连接器行为由用户全权负责**。本插件仅列出能力目录，用户对连接器的配置、启用、使用方式承担全部法律和安全责任。

### 法律风险警告
> ⚠️ **连接器可能被用于获取他人数据。请在使用前确保：**
> 1. 您对连接器接入的所有数据源**拥有合法访问权限**
> 2. 您获取的数据**不违反**任何保密协议、雇佣合同、服务条款或数据保护法律
> 3. 您所在司法管辖区的法律**明确允许**您通过连接器获取该类数据
> 4. 您已获得必要的内部授权（如公司管理层、法务部门、数据保护官的事先批准）

### 明令禁止的连接器用途
严禁通过连接器机制从事以下行为，违者可能承担民事、行政乃至刑事责任：
- 未经授权访问他人系统、数据库或文件
- 利用配置不当的 API 、弱口令或已知漏洞绕过访问控制
- 对第三方系统进行自动化大规模数据采集（爬取）而无合法依据
- 收集、处理或传输个人隐私数据而未遵守适用数据保护法规
- 将连接器获取的数据用于配置授权范围之外的用途

### 连接器安全最佳实践
1. **最小权限** — 只授予连接器完成任务所需的最小权限，不要使用全局 API Key
2. **访问控制** — 文件系统连接器应限定到特定目录，数据库连接器应使用只读账号
3. **日志审计** — 保留连接器使用日志，定期审查是否有异常访问模式
4. **凭据轮换** — 定期更换 API Key 和访问令牌
5. **及时撤销** — 调查结束后立即撤销连接器授权
