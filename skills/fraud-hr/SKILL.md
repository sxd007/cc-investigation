# 人力资源舞弊调查

适用于企业人力资源管理全流程中的舞弊调查。核心矛盾：**人为虚构或篡改 HR 相关数据，使公司为不存在的劳动、虚假的劳动或不合资格的人支付报酬**。

HR 舞弊属于 ACFE 资产挪用（Asset Misappropriation）— 薪资/工时舞弊（Payroll / Timesheet Schemes）类别，同时涉及腐败（Corruption）— 招聘舞弊（Recruitment Bribery）等跨类别行为。此类舞弊的特点是**涉及人数少但单案金额往往不大，但持续时间长、对组织公平感的侵蚀极大**。

> **与费用报销舞弊的关系**：HR 舞弊中的某些子模式（如福利滥用、虚假报销）与 `fraud-reimbursement` 有方法论重叠，但切入角度不同——HR 舞弊关注的是"**劳动关系和薪酬结构**中的虚假，而非费用报销申报中的虚假。

## When to Activate

- 工资总额与业务规模/人头数出现不匹配的异常增长
- 发现员工不存在或已离职但仍出现在薪资发放清单中
- 员工举报"有人不上班也领工资"或"有人帮别人打卡"
- 加班费/提成/奖金出现局部暴增，但对应的业务量无增长
- 招聘流程中出现异常——定向录用、面试走过场、推荐人关系异常
- 新员工入职后发现其资格/证书/履历与宣称不符
- 员工考勤数据与门禁记录、工作产出明显不匹配
- 福利使用数据异常（如长期病假但社交活动活跃、家属福利被冒用）

### 1. 虚假员工 (Ghost Employee)

**模式描述**：通过虚构不存在的员工添加至薪资系统，或将已离职员工保留在薪资清单中，由舞弊者冒领其工资。虚假员工是 HR 领域最经典的舞弊模式，通常需要 HR 或财务部门内部人配合才能实施。

**关键信号：**
- 薪资系统中的员工信息缺失必要资料（无身份证号、无银行账号、无紧急联系人）
- 员工记录的入职日期与薪资开始发放日期存在不可解释的间隔
- 离职员工的薪资未按时停发，或停发后又重新恢复
- 多个员工的薪资被发放到同一个银行账户（同一收款人领取多人工资）
- 员工薪资长期无人领取（支票/银行退票）
- 员工无社保/公积金缴纳记录但正常领取工资
- 员工在 HR 系统中"存在"但无人认识、无实际办公位、无工作产出
- 工资支付方式为现金或不可追踪的支付手段

**调查切入点：**
- 从薪资系统导出全量员工名单，与在职员工名单逐员比对
- 检查员工的人事档案完整度——缺少身份证、合同、学历证明等关键文件者优先排查
- 按银行账号聚合薪资发放记录，识别"一人多领"模式
- 核实社保/公积金缴纳人数与薪资发放人数的一致性
- 实地走访——声称在部门 X 的员工，部门同事是否认识该人
- 检查门禁系统、邮箱系统、OA 系统中是否有该员工的使用痕迹
- 追踪虚假员工薪资的资金走向——最终进入了谁的账户

### 2. 薪资操纵 (Payroll Manipulation)

**模式描述**：员工或 HR 系统管理员篡改薪资计算参数——包括虚报工时、篡改时薪/月薪、虚构加班（OT）、虚报考勤天数、人工调高奖金基数等，以获得超出应得额度的报酬。

**关键信号：**
- 个别员工的加班时长显著超出部门平均水平（月度加班超 50 小时/无休假记录）
- 加班申请与门禁记录/工作产出不一致——门禁记录显示早退但加班申报满时
- 薪资调整审批链缺失——薪资信息未经对应层级审批被修改
- 临时性薪资补发或调整发生在特定员工身上，且缺乏依据
- 同一岗位、同级别员工的薪资差异缺乏合理解释
- 手动调薪的频率和幅度异常（系统自动计算与手动调整个数之间的比例）
- 节假日/长假期间的加班申报密集

**调查切入点：**
- 交叉比对考勤记录、门禁记录、加班申请三者的时间一致性
- 按部门聚合分析加班分布——异常偏高或偏低的离群值重点排查
- 检查薪资系统变更日志——谁在何时修改了薪资参数
- 追踪手动薪资调整的审批链——是否有审批，审批人是否知情
- 邀约访谈——询问高加班时长的员工具体的工作内容和产出
- 比对假期期间的加班申报与实际的业务需求（如果公司放假期间无业务需求，则加班申报存疑）

### 3. 佣金/奖金舞弊 (Commission / Bonus Fraud)

**模式描述**：员工通过虚报业绩、篡改佣金计算参数、重复申报已结算业绩或与他人合谋完成虚假交易以获取佣金/奖金。常见于销售类岗位。

**关键信号：**
- 销售业绩在结算周期截止前集中暴增（"冲业绩"后大量退货）
- 员工的佣金/奖金收入与其实际贡献的业务价值不成比例
- 同一客户在短期内被不同销售人员重复计入业绩
- 出现"虚假订单"——客户无真实需求但下单以帮助销售达标
- 销售业绩集中在少数"大单"上，且这些订单的后续回款或交付出现异常
- 合谋信号：多名销售员同时集中向同一客户销售，或相互挂单/分单
- 销售业绩的认定时间点被刻意操控（将下一周期的业绩提前计入本期）

**调查切入点：**
- 分析佣金结算周期前后的订单模式——是否存在结算前集中创建、结算后退货的循环
- 按客户+销售员+时间三维度聚合，识别"合谋挂单"模式
- 追踪高额奖金对应的订单的真实性——核查物流签收、安装激活、客户验收
- 比对销售合约中实际的回款比例与佣金发放比例（公司可能按全额订单发佣但客户尚未全额付款）
- 检查退货/取消订单是否集中在佣金结算完成后
- 访谈客户——确认采购是否由销售人员促成或客户是否有真实需求

### 4. 招聘舞弊 (Recruitment Bribery / Hiring Fraud)

**模式描述**：HR 人员或业务部门面试官在招聘过程中收受候选人贿赂，为其提供内幕信息、定制面试题目、放宽录用标准或做出有利的面试评价。也可能表现为被要求"定向录用"某候选人（来自上级安排或关系户）。

**关键信号：**
- 面试评价与面试表现明显不符——低分者被录用，高分者被淘汰
- 职位描述仿佛为特定候选人"量身定制"（特定学历、特定年限、特定行业经验）
- 招聘过程中某候选人的面试轮次被简化或跳过（直接到终面）
- 候选人的来源渠道单一，且高度集中在特定猎头或推荐人
- 同一次招聘中，某候选人在各轮面试中的评分趋势出现矛盾（第一轮差→终面好）
- 某面试官的录用通过率显著高于其他面试官
- 录用人员的资质/能力与岗位要求明显不匹配

**调查切入点：**
- 调取招聘全流程记录——简历筛选 → 面试安排 → 面试评分 → 录用审批
- 对比各面试官的评分模式和录用倾向性——找出异常宽松的面试官
- 追踪候选人背景——与面试官或推荐人是否存在未披露的关联关系
- 分析推荐费支付模式——猎头/推荐人的收费是否与候选人入职成功率存在异常关联
- 核实录用人员的入职后绩效表现——若批量录用的人表现远低于平均水平，岗位录用标准可能被人为降低
- 面试评分表的原始打分对比——识别差异巨大的评分以排除"走过场"面试

### 5. 履历/资质造假 (Credentials Fraud / Resume Fraud)

**模式描述**：候选人在应聘时提交虚假的学历、资格证书、工作经历、薪资证明或背景信息，以获取本不具备资格的岗位或更高的薪酬。

**关键信号：**
- 学历信息无法在学信网等官方渠道验证
- 工作经历中的公司名称模糊/已注销/无法联系核实人
- 提供的证明人电话打不通、邮箱为个人邮箱而非公司邮箱
- 工作经历中出现难以解释的"空档期"或重合期（同一时间在两家公司全职工作）
- 项目经验描述使用通用技术术语但缺乏具体细节（可追问时无法自圆其说）
- 薪资证明的格式/盖章存在异常（可能是自制或非标准格式）
- 多个候选人的经历存在相同的夸大模式（暗示某个猎头批量协助造假）

**调查切入点：**
- 通过学信网/学位网等官方的学历认证平台核验学历信息
- 联系候选人声称的上家公司的 HR 或证明人核实工作经历
- 检查候选人在面试中提供的项目细节与简历的一致性——通过深度追问验证
- 在入职后设定考察期，观察候选人的实际能力是否与其宣称的资历匹配
- 如发现可疑，向前雇主核实候选人的职位、薪酬范围和离职原因
- 对照候选人提供的薪资流水或银行卡进账记录（如有授权）

### 6. 考勤舞弊 (Attendance / Timesheet Fraud)

**模式描述**：员工利用考勤制度的漏洞或通过代打卡、伪造门禁记录等方式虚报出勤，在未实际出勤的情况下领取全额的工资或津贴。

**关键信号：**
- 员工的考勤记录与其工作产出在逻辑上不匹配（出勤了但无成果）
- 同一部门多名员工的打卡时间高度同步（暗示可能存在代打卡或集体作弊）
- 某员工的考勤记录呈现高度规律性，从不迟到、从来不请假的"完美记录"
- 门禁记录显示进入办公区但工作电脑无操作痕迹
- 考勤记录与出差记录、请假记录出现重叠（同时在外出差又在本部打卡）
- 考勤记录被频繁手动修改，修改理由不合理

**调查切入点：**
- 交叉比对考勤打卡记录与门禁系统进出记录、电脑开机/系统登录记录
- 分析打卡 IP 地址——远程打卡是否来自非办公地点或 VPN 之后的地理位置异常
- 检查考勤系统日志——是否有考勤记录被手动添加或修改的历史
- 比对员工的实际产出与其出勤时间——是否"在位不在工"
- 访谈团队同事——了解被调查人的实际出勤情况
- 监控摄像头回放（如在合规前提下）——在关键时段确认实际出勤人员
- 分析跨部门协作记录——被调查人在出勤日的通讯/邮件量是否正常

### 7. 福利滥用 (Benefits Abuse / Fringe Benefit Fraud)

**模式描述**：员工通过虚假申报、冒用他人资格或重复报销等方式滥用公司提供的各项员工福利——如商业保险、体检、培训补贴、住房补贴、交通补贴、通讯补贴、子女教育补贴等。

**关键信号：**
- 某项福利的使用率远超公司平均水平或福利设定标准
- 家属福利被非家属人员使用（如将体检名额卖给外人）
- 同一福利被同一员工重复申报（利用不同渠道或不同审批人）
- 福利使用的时间、地点与员工的实际行程不匹配
- 员工在福利报销中提交的凭证存在伪造或变造的迹象
- 离职/退休人员的福利在某段时间内仍在持续发生
- 培训/教育类补贴申请但无对应的结业证明或学习记录

**调查切入点：**
- 提取全量福利使用数据，按福利类型+员工维度聚合，使用 Pareto 分析识别前 5% 的异常使用者
- 比对福利使用记录与员工的实际状态（离职/休假的员工是否仍在使用福利）
- 核验福利申请中附带的佐证材料——发票、收据、使用记录的真实性
- 对高频使用的家属福利进行电话回访确认——联系"受益人"确认是否真实享受
- 检查福利支出的资金流向——是否回到了员工或第三方的个人账户
- 对比同岗位、同级别员工的福利使用基准——识别离群者

## 调查方案设计要点

### 信号交叉验证

| 初始信号 | 通常涉及的模式 | 可能关联的其他领域 |
|---------|-------------|-----------------|
| 薪资名单与在职人数不匹配 | 虚假员工（优先） | 薪资操纵（可能配合修改系统） |
| 高额加班费/奖金 | 薪资操纵 / 佣金舞弊 | — |
| 面试评分异常 | 招聘舞弊 | 利益冲突（面试官与候选人有关联） |
| 员工考勤+门禁记录不一致 | 考勤舞弊 | — |
| 某类福利使用率远超平均 | 福利滥用 | 费用报销（如福利+报销两头拿） |
| 学历/资质查无实据 | 履历造假 | 招聘舞弊（面试官是否知情） |

### 数据分析切入点

| 分析维度 | 方法 | 预期发现 |
|---------|------|---------|
| **薪资账实核对** | 薪资发放名单 vs 在职人员名单，标记差异 | 虚假员工（多发）或遗漏（少发） |
| **银行账号聚合** | 按收款银行账号聚合，识别一卡多人 | 虚假员工收款归集 |
| **加班分布分析** | 按人/部门/时间聚合加班时数，计算标准差 | 异常加班者 |
| **佣金退货分析** | 佣金结算前后的退货率变化 | 虚假销售后退货 |
| **面试评分分析** | 面试官的评分均值、标准差、通过率 | 异常宽松/严格的面试官 |
| **考勤→门禁→工迹** | 考勤打卡+门禁记录+系统登录/邮件发送记录三维比对 | 只打卡不上班 |
| **福利使用 Pareto** | 按使用率聚合，取前 5% 深度核验 | 福利滥用高频违法者 |

### 数据来源分类

| 数据类别 | 内容 | 获取方式 |
|---------|------|---------|
| **系统内数据** | 薪资系统记录、HR 主数据、考勤记录、门禁记录、绩效记录、面试评分记录、福利使用记录 | 系统查询或导出（可在静默期完成） |
| **接触获取数据** | 员工原始档案（身份/学历/资质复印件）、面试评分表原件、劳动合同、对外背调记录 | 需接触 HR 部门，FIELDWORK 获取 |
| **公开查询数据** | 学信网学历验证、工商信息关联查询、公开社交平台核实、前雇主信息核实 | 公开渠道查询（可随时进行） |

### 调查原则

调查用到的系统内数据遵循 **双轨获取** 原则：

- **主路径（primary）**：通过系统接口或 SQL 脚本自动获取，结果可重复验证
- **备用路径（fallback）**：主路径不可用时，由调查员从系统手动提取，结果须人工核验
- **不可获取时**：标注数据缺口（evidence_gap），评估对 finding confidence 的影响

详细的通用调查原则（静默边界、缺口归因、信源前缀）参见 [`skills/investigation-foundation/SKILL.md`](../investigation-foundation/SKILL.md)。

## 分析辅助工具

以下工具类型可辅助加速 HR 舞弊相关数据分析。**它们不是必须的**——未配置时，由模型按上述方法直接完成分析。

### 数据库查询类 MCP（如配置）

- **能力：** 执行 SQL 查询、导出 HR 系统、薪资系统、考勤系统数据
- **辅助场景：** 薪资实发与应发人群交叉比对、银行账号重复检测、加班异常模式挖掘、面试评分一致性分析
- **不可用时：** 从 HR 系统手动导出数据，用电子表格分析

### 网络搜索类 MCP（如配置）

- **能力：** 公开信息检索、工商信息查询、学历验证
- **辅助场景：** 员工背景调查、学历真伪验证、候选人前雇主核实、面试官与候选人的关联关系查询
- **不可用时：** 浏览器手动查询学信网、工商网站、LinkedIn 等

### 推理验证类 MCP（如配置）

- **能力：** 分步逻辑推演、多路径比对
- **辅助场景：** HR 数据交叉比对逻辑设计、薪资资金流向推演、招聘异常模式推演
- **不可用时：** 由模型自行完成推演，输出推演过程供审查
