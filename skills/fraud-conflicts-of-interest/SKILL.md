# 利益冲突舞弊调查

适用于企业内部涉及利益冲突的调查场景。核心矛盾：**员工在商业决策中的个人利益与公司利益之间的冲突**。

利益冲突不属于传统的资产挪用，而是员工利用决策权或影响力为关联方输送利益，间接损害公司利益。属于 ACFE 腐败（Corruption）— 利益冲突（Conflicts of Interest）类别。此类舞弊往往最难发现，因为交易本身通常是真实的，问题在于**决策者未披露其与交易对手的关联关系**。

## When to Activate

- 收到员工利益冲突相关的举报（员工在外开办公司、关联交易未披露等）
- 采购/招投标中出现异常偏向特定供应商的模式
- 供应商背景调查发现与公司员工存在未披露的关联关系
- 员工资产/消费水平与合法收入明显不匹配
- 发现回扣或贿赂线索
- 需要设计利益冲突调查方案
- 并购/投资尽职调查中发现可疑关联方

## 五种舞弊模式

### 1. 采购冲突 (Purchasing Schemes)

**模式描述**：员工在采购决策中偏向其拥有直接或间接经济利益的供应商，通常以高于市场价采购或授予独家合同，关联供应商通过回扣、分红或雇佣亲属等方式回报员工。

**关键信号：**
- 供应商通过"邀请招标"或"单一来源"方式获得合同，而非公开竞争
- 供应商报价虽"最低"但仅比次低高 0.1%-0.5%（卡线中标）
- 采购项目被刻意拆分为多个小合同以规避审批限额
- 供应商历史较短、注册资本低、团队规模与合同金额不匹配
- 供应商提供的产品或服务价格持续高于市场平均水平
- 供应商与公司员工存在同姓名、同地址、同联系方式等关联线索

**调查切入点：**
- 核验供应商工商信息：股东、董事、监事是否包含公司员工或其亲属
- 绘制供应商实控人关系网，识别与员工的隐性关联
- 分析中标率、中标价格分布，评估是否存在偏向性
- 追踪付款资金流向（供应商→关联人→员工/亲属的回流路径）
- 核查员工个人账户交易记录（如有合规授权）

### 2. 销售冲突 (Sales Schemes)

**模式描述**：员工将公司客户/业务机会私下转移给其拥有权益的外部实体，或向客户提供不当折扣/信贷条件以换取个人利益。与采购冲突相反，利益输送方向是"向外"的。

**关键信号：**
- 大客户突然流失至某特定渠道或代理商
- 客户被引导与特定代理商签约而非直接与公司交易
- 给予特定客户异常高的折扣率或宽松的信用期
- 新渠道/代理商的创始人或核心团队与公司员工存在历史关联
- 客户投诉"被转介"给特定服务商
- 员工在离职前密集将客户关系转移至某外部公司

**调查切入点：**
- 比对客户流失时间线与员工行为（是否在流失前有异常操作）
- 核验接收流失业务的代理/渠道是否与员工存在关联关系
- 分析折扣审批记录，识别对特定客户的异常让步
- 追踪客户投诉记录中对"转介""指定"的抱怨
- 借助员工个人社交网络发现隐性关联

### 3. 裙带关系 (Nepotism / Cronyism)

**模式描述**：员工利用职权为亲属、朋友或关系人提供不当的雇佣、晋升、合同或商业机会。这类行为有时披着合规外衣（程序完备但实质不公）。

**关键信号：**
- 员工亲属或其关联人在公司任职或在同一业务链条上
- 特定部门存在非正常的"家族聚合"现象
- 员工的亲属供应商在无实质竞争情况下持续获得合同
- 招聘流程中特定候选人被"定向"录用（面试走过场、定制 JD）
- 员工下属或同事与其存在未披露的私人关系
- 同一部门的请假/出差模式高度一致（暗示私下关系）

**调查切入点：**
- 核验员工亲属关系声明（如有）的完整性和真实性
- 使用工商信息查询工具筛查员工亲属是否担任供应商股东/高管
- 分析招聘档案：审批链、面试评价、候选人来源（是否由员工推荐）
- 对比亲属供应商与其他供应商的中标率、价格水平、服务质量
- 覆盖多层间接关系（姻亲、同学、前同事等），不限于直系亲属

### 4. 双重角色 (Dual Role Conflict)

**模式描述**：员工同时在公司和对立方任职或拥有权益，使其处于无法同时忠于双方的境地。典型的场景包括：在客户公司兼职、在与公司交易的机构中持股、担任竞争对手的顾问或董事。

**关键信号：**
- 员工在外担任其他公司的法定代表人、董事、监事或高级管理人员
- 员工个人银行流水显示来自疑似竞争对手或供应商的定期款项
- 员工在保密信息知悉范围后，竞争对手的市场反应异常迅速
- 员工在外有注册公司且经营范围与公司业务重叠
- 员工长期不在岗或工作时间"碎片化"（同时管理两家公司）
- 员工在竞业限制期内加入竞争对手或在客户处任职

**调查切入点：**
- 利用工商查询系统筛查员工兼职/持股情况
- 比对外部商事登记信息与员工申报信息的一致性
- 分析员工工作邮件、日程中涉及外部企业的线索
- 排查员工个人公司的客户、供应商与公司客户的交叉关系
- 追溯竞争对手获得公司内部信息的时间链

### 5. 回扣关联 (Kickback Arrangements)

**模式描述**：供应商或客户向决策员工支付回扣以换取商业利益。回扣可以是现金、实物、服务、旅行、股权或其他任何形式的有价物。此模式常与采购冲突、销售冲突相伴出现。

**关键信号：**
- 供应商或客户的资金流向存在无法解释的"中介方"账户
- 供应商实控人与员工有共同投资、消费分担或私人借款记录
- 员工与供应商人员存在大额、频繁的个人资金往来
- 回扣金额与订单金额存在可计算的"提成比例"关系
- 员工生活方式出现与其合法收入不匹配的改善（新房、新车、奢侈品）

**调查切入点：**
- 追踪资金从公司→供应商→（可能的中间人）→员工的完整路径
- 分析员工私人消费、资产购置时间与供应商付款时间的关联性
- 识别第三方"居间人"或"咨询公司"——回扣常通过这些壳公司洗白
- 比对公司给供应商的付款时间与员工个人的消费/储蓄时间节点
- 在访谈中设置"意外问题"（如突然问及其对特定供应商的日常接触）

## 调查方案设计要点

### 信号交叉验证

利益冲突的各种模式经常相互关联，一个模式往往牵出其他模式：

| 初始信号 | 通常伴随的模式 | 可能涉及的其他领域 |
|---------|-------------|-----------------|
| 供应商中标模式异常 | 采购冲突 + 回扣关联 | 费用报销（合谋后报销作假） |
| 员工与企业主有私人关系 | 裙带关系 + 采购冲突 | — |
| 客户流失至特定代理商 | 销售冲突 | 渠道舞弊（窜货+虚报终端） |
| 员工在外注册公司 | 双重角色 + 销售冲突 | — |
| 员工生活方式异常 | 回扣关联 +（采购/销售冲突） | — |

### 关联关系调查方法

利益冲突调查的核心挑战是发现"隐藏的连接"。推荐结构化方法：

| 方法 | 适用阶段 | 说明 |
|------|---------|------|
| **工商交叉查询** | 静默期 | 员工 + 亲属姓名 vs 供应商/客户/代理商的工商数据交叉比对 |
| **社交图谱** | 静默期+接触期 | 基于公开信息构建员工→实体间的关系网络 |
| **资金流向追踪** | 静默期（有授权） | 银行流水中的异常收款、定期汇款、与供应商/客户的私人资金往来 |
| **通讯数据分析** | 静默期（有授权） | 邮件、即时通讯中涉及外部公司/个人利益的指示性信息 |
| **时间线关联** | 全阶段 | 员工的关键决策时间 vs 关联方成立时间/获单时间/资金往来时间 |

### 证明难点

利益冲突的证明通常比直接挪用更难，需要注意：

1. **意图要件**：员工未披露利益冲突未必等于舞弊——需要证明其**明知**有利益关系但**故意**未披露，且交易**不公允**
2. **替代理论**：被调查人可能辩称"供应商是最优选择，无关个人关系"——因此需同时证明（a）存在利益关系，且（b）该关系影响了决策
3. **关联关系发现**：间接关系（姻亲、前同事、代持）难以通过简单工商查询发现，需要多源数据交叉

### 数据来源分类

| 数据类别 | 内容 | 获取方式 |
|---------|------|---------|
| **系统内数据** | 合同/订单记录、供应商主数据、招投标记录、折扣审批记录 | 系统查询或导出（可在静默期完成） |
| **接触获取数据** | 员工入职申报的利益冲突声明、供应商调查问卷、工作日程 | 需接触 HR/业务部门，在 FIELDWORK 获取 |
| **公开查询数据** | 工商注册信息、股权结构、企业图谱、关联方诉讼记录、社交媒体 | 公开渠道查询（可在静默期完成） |

### 调查原则

调查用到的系统内数据遵循 **双轨获取** 原则：

- **主路径（primary）**：通过系统接口或 SQL 脚本自动获取，结果可重复验证
- **备用路径（fallback）**：主路径不可用时，由调查员从系统手动提取，结果须人工核验
- **不可获取时**：标注数据缺口（evidence_gap），评估对 finding confidence 的影响

详细的通用调查原则（静默边界、缺口归因、信源前缀）参见 [`skills/investigation-foundation/SKILL.md`](../investigation-foundation/SKILL.md)。

## 分析辅助工具

以下工具类型可辅助加速利益冲突调查的数据分析。**它们不是必须的**——未配置时，由模型按上述方法直接完成分析。

### 数据库查询类 MCP（如配置）

- **能力：** 执行 SQL 查询、导出采购/招投标/供应商明细数据
- **辅助场景：** 批量比对供应商工商数据与员工信息、统计中标率异常、追踪付款时间线
- **不可用时：** 从系统手动导出数据，用电子表格或文本编辑器分析

### 网络搜索类 MCP（如配置）

- **能力：** 公开信息检索、工商信息查询
- **辅助场景：** 员工及亲属的企业关联关系查询、供应商背景调查、社交媒体分析
- **不可用时：** 浏览器手动查询工商网站、搜索引擎检索

### 推理验证类 MCP（如配置）

- **能力：** 分步逻辑推演、多路径比对
- **辅助场景：** 实控人关系网络推演、资金流向路径推演、时间线交叉验证
- **不可用时：** 由模型自行完成推演，输出推演过程供审查
