# 投标操纵调查

适用于招投标和竞争性报价过程中的舞弊调查。核心矛盾：**投标竞争被合谋行为破坏，采购方无法获得真实的市场竞争价格**。

投标操纵属于 ACFE 腐败（Corruption）类别，在世界各国也是反垄断执法（Antitrust / Competition Law）的重点领域。此类舞弊不要求招标方员工一定有个人利益——投标商之间可以独立合谋（"纯粹围标"），也可与招标方员工内外勾结（"串标"）。

> **与采购舞弊的关系**：采购舞弊（`fraud-procurement`）覆盖从需求到付款的**全生命周期**，投标操纵是其核心环节之一。但投标操纵也可独立存在——如政府招投标、PPP 项目、特许经营权竞标、线上拍卖等非企业采购场景。两个技能配合使用可获得完整覆盖。

## When to Activate

- 供应商/投标人投诉招投标不公或"走过场"
- 中标结果出现高度规律性（"轮流坐庄"、中标份额稳定）
- 投标人数量刚好满足最低开标条件（"凑人头"）
- 投标文件存在无法解释的相同特征（格式、错误、排版一致）
- 同一项目投标人之间存在异常的资金往来或商业关系
- 投标人举报竞争对手私下接触或存在"串通"行为
- 内审发现招标流程异常（单一来源比例过高、投标周期过短、评分异常）
- 政府执法机构对行业发起反垄断调查时

## 投标操纵的四种基本形态

投标操纵依据**合谋范围**分为四种基本形态，在复杂案件中往往以组合形式出现：

```
                       合谋范围
                   ┌─────────────┐
                   │ 横向合谋     │ 纵向合谋
                   │ (投标方之间) │ (招标方+投标方)
        ┌─────────┼─────────────┼─────────────┐
        │         │ 围标/轮流   │ 串标/运作   │
        │ 供应商  │ 市场划分    │ 定向招标    │
        │ 主动    │ 转包回补    │ 信息泄露    │
        │         │ 陪标报价    │ 参数定制    │
        ├─────────┼─────────────┼─────────────┤
        │         │             │             │
        │ 招标方  │   —         │ 收受回扣    │
        │ 主动    │             │ 规避集采    │
        └─────────┴─────────────┴─────────────┘
```

## 八种操纵模式

### 1. 压标 (Bid Suppression)

**模式描述**：潜在投标人之间约定某些方不参与投标或撤回已递交标书，以减少竞争、确保指定方中标。压标是围标中最隐蔽的形式——表面上"没有人阻止你投标"，但行业默契使竞标者自行退出。

**关键信号：**
- 有资质/有历史投标记录的公司突然未参与投标
- 资格预审后投标人数量锐减
- 有投标人购买了标书（记录在案）但未递交标书
- 投标人退出投标后即成为中标方的分包商
- 行业中主要竞争对手在同一项目上"默契"缺席

**调查切入点：**
- 对比历史投标人清单与本次实际投标人清单，识别"异常缺席"者
- 访谈未投标但有资质的企业，了解不投标的原因
- 追踪缺席者与中标方之间是否有分包合同或资金往来
- 检查资格预审后的通讯记录，寻找"劝退"的证据
- 关注有投标意向但最终未递交标书的公司（"买了标书不投"）

### 2. 陪标 / 掩护性投标 (Complementary Bidding / Cover Pricing)

**模式描述**：有意中标方安排其他投标人提交掩护性标书——报价明显偏高、包含特殊条件或明知不合要求——使中标方在形式上"经充分竞争胜出"。

**关键信号：**
- 陪标方报价与中标方差距显著（高 5%-20%），且无合理成本解释
- 陪标方报价为整位数（如 5,000,000、2,380,000）而中标方报价精确到个位
- 陪标方标书中出现与中标方相同的排版错误、拼写错误、计算公式错误
- 陪标方标书使用与中标方相同的模板、文件属性、字体配置
- 陪标方标书缺少关键文件（如资质证书、业绩证明），"故意"不合格
- 陪标方标书包含明显不利的条件（如超长付款周期、缩短质保期）

**调查切入点：**
- 提取所有标书的文件元数据（创建时间、作者、最后保存者、版本历史、文件路径），寻找共同出处
- 对比标书中的错误模式——同型错误是串通的有力证据
- 分析陪标方报价的计算逻辑——离奇的不合理报价暗示非真实意图
- 检查陪标方是否在投标后通过其他方式获得补偿（分包、转包、咨询费）
- 提取投标保证金/保函的出票方——如果多家保函来自同一银行同一天，高度可疑

### 3. 轮标 (Bid Rotation)

**模式描述**：投标商之间约定轮流中标，确保各家在长期中获取大致均等的市场份额。轮标通常伴随明确的"补偿机制"——轮流方案和补偿金额。

**关键信号：**
- 同一供应商群在不同项目中轮流中标，模式有规律可循
- 中标分布均匀——各家市场份额接近甚至相等（"公平分享"）
- 同一供应商在不同项目中的报价波动极大（在轮到的项目上报价正常，在陪标项目上报价偏高）
- 中标方在报价、技术评分上也"轮流"——这次技术高分，下次技术低分但价格低
- 长期中标记录的统计学分布不服从随机性

**调查切入点：**
- 按时间序列排列中标记录，检查是否存在周期性模式（如"今年 A 明年 B 后年 C"）
- 计算各供应商的市场份额 CV（变异系数）——过于均匀是轮标的特征
- 按项目类型/区域/金额分层分析中标分布模式
- 进行马尔可夫链分析——上一次中标方与下一次中标方的关系是否存在规律
- 检查中标方的"回报"行为——本轮中标方是否在后续项目中补偿陪标方

### 4. 市场划分 (Market Allocation)

**模式描述**：投标商之间通过划分客户、地域、产品或项目类型来分配市场，各自在自己的"领地"内独占。

**关键信号：**
- 特定供应商只出现在特定区域/客户/产品类别的投标中
- 跨区域投标的供应商从不"入侵"同行的领地区域
- 某供应商在其"领地"内中标率极高，但在其他区域从不/极少投标
- 多家供应商的投标范围存在"边界默契"，不相互覆盖
- 客户报告说某些供应商相互推诿或"建议找另一家"

**调查切入点：**
- 绘制供应商投标的地理/客户/产品覆盖热力图，识别"领地边界"
- 分析各供应商在不同区域/客户的中标率差异——正常竞争应有交叉
- 访谈客户——是否曾被特定供应商"推荐"给其他供应商
- 检查供应商之间是否存在"不竞争"的书面或口头协议
- 对比各供应商在同一区域的报价水平——在自己的领地内报价通常偏高（缺乏竞争）

### 5. 转包回补 (Subcontracting Compensation)

**模式描述**：中标方将部分合同分包给曾配合围标的未中标方，或向其支付补偿金，作为不投标或陪标的回报。这是围标合谋的"结算机制"。

**关键信号：**
- 中标方与未中标方之间存在分包合同或咨询协议
- 分包金额大致等于围标补偿的期望值
- 未中标方的"咨询费"或"服务费"与中标金额存在可计算的固定比例
- 分包/补偿发生在投标结束后的短时间内
- 中标方与配合方之间的资金流转模式呈规律性
- 未中标方收到补偿后不再参与之后的投标（"已完成结算"）

**调查切入点：**
- 追踪中标方与未中标方之间在投标后的资金往来
- 检查中标方的分包合同——分包方是否包括本次未中标方
- 分析"咨询协议"或"服务合同"的真实商业目的和实际交付
- 追踪补偿金额+分包合同的组合价值，判断是否构成"公平回报"
- 访谈分包商的工作内容——是否存在"只收费不干活"的分包安排

### 6. 信息泄露 (Information Leaking)

**模式描述**：招标方员工（或第三方）将投标底价、竞争对手报价、评审标准、评审进度等信息泄露给特定投标人，使其获得不公平竞争优势。

**关键信号：**
- 某投标人的报价精准卡线——正好是次低价的 99.5% 或略高于底价
- 某投标人的技术方案高度契合尚未公开的评审偏好
- 中标方在开标前即已开始备货或投入（表明提前知道中标结果）
- 投标人报价与预算金额/标底的差异极小
- 有投标人投诉"我的报价被泄露"
- 投标人在答疑环节提出的问题暗示其掌握非公开信息

**调查切入点：**
- 追踪标底/预算金额的知悉范围和传播路径
- 检查招标文件访问记录——谁在何时打开过文件
- 对比各投标人的报价与标底的接近程度——异常接近者可能获取了信息
- 追踪中标方在投标前的异常行为（提前采购、提前招聘、前置投入）
- 检查招标方员工在投标期间的通讯记录与投标人之间的联系
- 通过"蜜罐"信息陷阱——在招标文件中设置虚假条款，观察是否有多家投标人以相同方式回应

### 7. 招标参数操控 (Tender Specification Rigging)

**模式描述**：招标方员工为特定投标人"量身定制"招标参数、评分规则或评审标准，形式上合规但实质上排除竞争。此模式不限于采购——也可用于特许经营权、资质许可等非采购场景。

**关键信号：**
- 技术参数中存在只有特定品牌/型号才能满足的排他性要求
- 评分标准中主观权重异常偏高（"行业经验""团队实力"等难以量化比较的维度）
- 资质要求设置时间极短（如"须有同类项目经验 5 年"但某特定投标人刚好成立 5 年）
- 招标公告后只有一家或极少数投标人符合资格
- 评审标准在发标后被修改，且修改方向有利于特定投标人
- 评分结果与评分标准的权重在逻辑上不匹配

**调查切入点：**
- 比对本次招标参数与同类项目历史参数的差异，识别异常的特设要求
- 检查参数制定者的背景——是否与中标投标人存在过往合作或关联关系
- 邀请独立专家评估参数的合理性和排他性
- 审查评分表的原始打分情况，是否存在对某些投标人的系统性偏高或偏低
- 追踪参数设定过程中的修改记录和审批历史

### 8. 围标未遂与举报报复 (Failed Collusion & Retaliation)

**模式描述**：供应商拒绝参与围标后被其他合谋方报复，或围标破裂后参与方相互举报。此类案件的信号往往来自行业内部"叛变者"。

**关键信号：**
- 某供应商在拒绝参与围标后连续失去已有业务
- 新进入市场的供应商报价远低于"市场价格"（打破既有合谋）
- 匿名举报来自行业内人士（知情人报料的典型特征）
- 行业内出现集中性举报——围标破裂后的"互相告发"
- 某供应商在与同行交流后大幅调整报价

**调查切入点：**
- 特别保护举报人（此类举报人面临真实报复风险）
- 追踪拒绝参与围标者的后续遭遇——是否被同行排挤或客户流失
- 分析新进入者报价与行业平均水平的差异——大幅低于均值是打破合谋的典型信号
- 对比合谋破裂前后的价格变化，量化合谋给采购方造成的损失
- 建立"吹哨人"保护性沟通渠道

## 调查方案设计要点

### 信号交叉验证

投标操纵的多种模式往往同时出现，构成一个完整的合谋闭环：

| 初始信号 | 通常伴随的模式 | 合谋阶段 |
|---------|-------------|---------|
| 投标人数量异常偏少 | 压标 + 市场划分 | 合谋形成阶段 |
| 标书存在相同错误/格式 | 陪标 + 轮标 | 合谋执行阶段 |
| 中标模式呈现规律性 | 轮标 + 转包回补 | 合谋结算阶段 |
| 报价异常接近标底 | 信息泄露 + 陪标 | 合谋+内部人 |
| 供应商不跨区域投标 | 市场划分 + 轮标 | 合谋制度化 |

### 反垄断调查思维

投标操纵同时也是反垄断执法问题。调查者需要从两个维度思考：

| 维度 | 采购方关注 | 反垄断机构关注 |
|------|-----------|-------------|
| **损害** | 公司多付了多少钱 | 社会福利损失和市场扭曲 |
| **证据重点** | 内部人参与了没有 | 投标商之间合谋的行为 |
| **证明标准** | 优势证据（Preponderance） | 排除合理怀疑（刑事标准） |
| **处理方式** | 内部追责+终止合同 | 行政罚款+刑事追诉 |
| **配合策略** | — | 在反垄断调查中配合可获宽大处理（Leniency） |

### 数据分析切入点

| 分析维度 | 方法 | 预期发现 |
|---------|------|---------|
| **中标集中度** | 计算品类/区域层面的 Herfindahl-Hirschman 指数 | 市场被少数供应商控制的合谋格局 |
| **报价离散度** | 分析同一项目各投标人报价的标准差 | 标准差过小→围标；过大→陪标 |
| **中标轮换指数** | 按时间序列建模中标顺序的随机性 | 非随机性→轮标 |
| **跨项目关联** | 构建"投标人×项目"共现矩阵 | 同一群投标人在多个项目上同时出现→固定圈子 |
| **价格趋势断点** | 识别价格水平的突然变化时间点 | 合谋形成/破裂的时间节点 |
| **投标人行为变化** | 追踪单个投标人的投标行为随时间的变化 | 新进入者打破合谋，或参与者逐渐形成默契 |
| **文件元数据聚类** | 提取标书元数据做谱系聚类 | 同一源头制作的标书群组 |

### 证据链构建

投标操纵调查的证据需要分层推进：

| 层面 | 证据类型 | 获取难度 |
|------|---------|---------|
| **经济证据**（第一层） | 价格异常、中标模式、市场结构 | 低——数据分析即可得出 |
| **通信证据**（第二层） | 投标人之间的邮件、电话、会议记录 | 中——需合法调取 |
| **文件证据**（第三层） | 标书元数据、合同文件、资金往来记录 | 中——系统内可获取 |
| **口供证据**（第四层） | 当事人陈述、举报人证言、宽大申请 | 高——需访谈/主动配合 |

> 经济证据虽不能单独定案，但可以：（1）指导搜索通信和文件证据的方向；（2）在访谈中用于"意外质询"——让被访谈人解释异常数据；（3）在缺少直接证据时形成间接证据链。

### 数据来源分类

| 数据类别 | 内容 | 获取方式 |
|---------|------|---------|
| **系统内数据** | 招投标记录、供应商/投标人主数据、标底/预算记录、评分表、审批链、合同记录 | 系统查询或导出（可在静默期完成） |
| **接触获取数据** | 标书原件（纸质/电子版及其元数据）、评审原始打分表、投标人通讯记录 | 需接触招标部门/存档部门，FIELDWORK 获取 |
| **公开查询数据** | 投标人工商登记信息、股权结构、关联图谱、历史中标公告、行业反垄断调查记录 | 公开渠道查询（可在静默期完成） |

### 调查原则

调查用到的系统内数据遵循 **双轨获取** 原则：

- **主路径（primary）**：通过系统接口或 SQL 脚本自动获取，结果可重复验证
- **备用路径（fallback）**：主路径不可用时，由调查员从系统手动提取，结果须人工核验
- **不可获取时**：标注数据缺口（evidence_gap），评估对 finding confidence 的影响

详细的通用调查原则（静默边界、缺口归因、信源前缀）参见 [`skills/investigation-foundation/SKILL.md`](../investigation-foundation/SKILL.md)。

## 分析辅助工具

以下工具类型可辅助加速投标操纵数据分析和关联发现。**它们不是必须的**——未配置时，由模型按上述方法直接完成分析。

### 数据库查询类 MCP（如配置）

- **能力：** 执行 SQL 查询、导出投标/中标/供应商明细数据
- **辅助场景：** 中标模式分析、报价离散度计算、供应商共现矩阵构建、文件元数据聚类
- **不可用时：** 从招投标系统手动导出数据，用电子表格 + 文本编辑器分析

### 网络搜索类 MCP（如配置）

- **能力：** 公开信息检索、工商信息查询、历史中标公告检索
- **辅助场景：** 投标人关联关系查询、人员交叉任职排查、行业反垄断记录检索
- **不可用时：** 浏览器手动查询工商网站、招投标公告平台、搜索引擎检索

### 推理验证类 MCP（如配置）

- **能力：** 分步逻辑推演、多路径比对
- **辅助场景：** 投标人合谋网络推演、轮标模式识别、经济证据→访谈策略的转化
- **不可用时：** 由模型自行完成推演，输出推演过程供审查
