# 伪造印章调查

适用于企业公章、合同章、财务章、法人章、授权章等各类印章被伪造、变造、盗用或冒用的调查。核心矛盾：**印文的表面真实性与盖章行为的实际授权性之间的差异**。

伪造印章不是独立的舞弊类型，而是**实现其他舞弊的手段**——它是采购舞弊中的"合同造假"、HR 舞弊中的"离职证明"、费用报销中的"虚假审批"的常见工具。因此伪造印章调查往往不单独发生，而是作为其他案件调查中的支援环节被触发。

> ⚠️ **本技能聚焦调查员层面的特异性认知**——即识假、追踪、取证的方法论。印章真伪的**技术鉴定**（显微镜比对、光谱分析、电子印章加密核验）须委托专业鉴定机构。用户可按需补充鉴定脚本至 `references/scripts/` 目录。

## When to Activate

- 对方拒绝提供合同原件，仅提供复印件/扫描件/拍照件
- 合同/文件上的印章印文与存档印模明显不同
- 印章的尺寸、字体、排版等视觉特征与真实印章存在差异
- 同一份文件上各印章的油墨新旧程度不一致（暗示不同时间加盖后合成）
- 电子印章的加密验证失败或证书信息异常
- 对方在质疑印章真实性后态度变得回避或采取拖延策略
- 印章号码（公安机关备案编号）在官方查证系统中不存在或属于其他单位
- 多份文件上的印章看似相同但细部存在不可解释的差异（"套印"嫌疑）

## 四种伪造形态

### 1. 私刻 (Forged / Counterfeit Chop)

**模式描述**：未经授权私自刻制与公司公章外观相似或完全相同的印章。私刻可以是完全仿制，也可以是"做旧"仿冒老版本印章。私刻印章通常用于独立制作虚假文件。

**关键信号：**
- 印文边缘模糊、不清晰（刻制精度不足，尤其是仿制铜章时）
- 字体与真实印章不一致（仿宋 vs 宋体 vs 楷体的细微差异，缺笔少划）
- 印章尺寸与真实印章相差 1mm 以上（手工描摹复刻时难以精确还原）
- 印章编号与公安备案编号不符或编号格式不符合当地规范
- 私刻印章的"做旧"痕迹不自然——油墨堆积或人工磨损呈均匀分布（自然磨损是不均匀的）

**调查切入点：**
- 调取公司在公安机关的预留印鉴卡或公证章模进行叠图比对
- 测量印文的精确尺寸（直径、文字间距、边框宽度）
- 检查印章编号的合法性——通过公安公章备案系统查证
- 比对多份已确认真实文件上的印章，确认是否有多版本差异
- 询问经办人印章的来源——"是谁在何时何地盖的？"

### 2. 变造 (Altered / Modified Chop)

**模式描述**：在真实印章的印文基础上进行局部修改——如篡改公司名称（"有限公用"改成"有限公司"）、改动合同金额数字处的骑缝章、遮盖部分文字后复印再盖章等。

**关键信号：**
- 印文中存在涂改、刮擦、覆盖、填补的物理痕迹
- 印文的文字间距不自然（修改后的文字与原始文字的间距不一致）
- 变造区域附近的文字在排版上存在偏移（如某字的位置略高于同行其他字）
- 印文覆盖区域存在纸张纤维的破坏或残留胶带痕迹
- 扫描件/复印件上的印章区域有拼接缝或色调不一致

**调查切入点：**
- 调取原件（而非复印件）在放大镜下检查印文区域
- 使用侧光检查印文表面是否有凹凸不平的修改痕迹
- 通过透光检查纸张厚度是否有局部异常（贴补）
- 对比同一印章在不同文件上的尺寸——变造可能导致缩放比例不一致
- 检查骑缝章与合同页的完整性——是否存在页被替换后重新骑缝

### 3. 盗用 (Unauthorized Use of Genuine Chop)

**模式描述**：真实印章被未经授权的人在未经批准的场合使用——如员工趁保管人不在时私盖、保管人越权盖章、印章被调包后盗用。盗用的是**真实的印章**，因此印文本体无异常，问题在于**授权的真实性**。

**关键信号：**
- 印章本体的印文与存档印模完全一致（真章），但文件内容可疑
- 盖章记录显示印章在非工作时间被取出使用
- 用印审批单上的审批人签名看起来是由同一人签署的
- 保管人报告的盖章时间与用印申请记录不一致
- 同一时间段内印章出现在不同地点的文件上（如上海总部和北京分公司同时使用同一公章）

**调查切入点：**
- 调取用印登记簿或电子用印系统的操作日志——谁、何时、因何事调用印章
- 核实用印审批单的真实性——签名是否真实、审批流程是否完整
- 访谈保管人——盖章时的具体情况（谁来盖的、盖了什么、是否有异常）
- 比对疑似盗用文件上的盖章时间与保管人出勤、休假记录的重合性
- 检查印章存放地点的监控记录（如有）——是否有未授权人员接触印章

### 4. 冒用 (Misrepresentation / PS Composite)

**模式描述**：并非伪造印章本身，而是伪造含有印章印文的**完整文件**——通过彩色打印、扫描再制、PS 合成等方式制作带有伪造印文的文件，而非在纸质文件上加盖物理印章。

**关键信号：**
- 文件上的印章颜色过于均匀或出现印刷品的网点（用放大镜观察）
- 印章红边与文件其他内容的黑色文字在色调上存在不自然的融合
- 印章印文在文件中处于文字层的"下方"（真章盖在文字上方，但 PS 合成时往往印章在下文字在上）
- 文件的 PDF 属性/元数据中含有 Photoshop 等图像编辑软件的创建记录
- 印章区域放大后出现锯齿边缘（位图放大失真）而非真实盖印的油墨边缘
- 文件中印章的透明度或混合模式不自然（与真实盖印的油墨覆盖性不同）

**调查切入点：**
- 索要纸质原件或原始 PDF（非扫描件）——冒用通常只能提供复印件/扫描件
- 检查 PDF 的元数据——是否有 Photoshop/GIMP 等编辑软件的痕迹
- 检查印章区域的图层分析——在 PDF 中印章是否是一个独立的图像对象（Object）
- 将文件放大至 300%+ 观察印章边缘——真实印章是油墨扩散的随机纹理，印刷/PS 是规律性网点
- 检查文件中文字的连续性——PS 合成可能导致印章覆盖部分文字被"吃掉"
- 追溯文件电子版的来源路径——是谁通过什么渠道发来的

## 调查方案设计要点

### 真伪判断流程

面对可疑印章文件，建议使用以下结构化推理流程：

```
第一步：确定文件类型
  ├── 有纸质原件 → 进入第二步
  └── 只有复印件/扫描件 → 标记"证据降级"并进入第三步

第二步：物理检验（有原件时）
  ├── 目视检查（尺寸、字体、比例、颜色）
  ├── 放大镜检查（边缘、油墨分布、纸张纤维）
  ├── 侧光检查（涂改、刮擦、补纸）
  └── 紫外光检查（荧光反应、纸张差异）

第三步：内容关联检验（所有情况）
  ├── 印章+文字关系（谁盖在谁上面？全盖还是半盖？）
  ├── 印章+日期关系（用当前版本的印章盖过去的文件？）
  ├── 印章+使用范围（这个章能用于此类文件吗？）
  └── 印章+审批链（有对应的用印申请吗？）

第四步：来源追溯
  ├── 电子版来源追踪（邮件/IM/系统上传）
  ├── 经办人访谈（谁准备、谁打印、谁盖章）
  └── 交易对手核实（对方是否收到文件？盖章窗口期是否合理？）

第五步：专业鉴定（如必要）
  ├── 委托司法鉴定机构进行印文同一性鉴定
  ├── 委托电子数据鉴定进行电子印章真伪核验
  └── 鉴定脚本见 references/scripts/（用户补充）
```

### 双轨取证策略

| 证据线 | 内容 | 说明 |
|-------|------|------|
| **印章本体证据** | 可疑印章文件 + 真实印模样本 + 公安备案记录 + 鉴定意见 | 证明"章是假的" |
| **行为证据** | 用印记录 + 审批链 + 经办人访谈 + 文件流转路径 | 证明"运用是未经授权的" |
| **目的证据** | 文件签署背景 + 交易对手证言 + 资金流向 | 证明"伪造的目的" |

三条证据线各自独立、互相补充。即使第一条"无法鉴定"（如只有扫描件），行为证据 + 目的证据也可能形成完整证据链。反之，鉴定了印章造假但无法证明使用目的，也需要行为证据补强。

### 文件层级区分

在不同的文件呈现形式上，调查者能做的判断深度不同：

| 文件形式 | 可做的判断 | 不可做的判断 |
|---------|-----------|-------------|
| **纸质原件** | 物理特征检验、叠图比对、油墨分布、压痕深度 | 无法独立判断电子印章真伪 |
| **高分辨率扫描件（600dpi+）** | 尺寸比例、字体细节、印章+文字关系、元数据检查 | 无法做物理特征检验（油墨、纸张） |
| **低分辨率扫描/拍照件（<300dpi）** | 宏观特征（印文位置、内容、使用逻辑） | 无法做细部比对，依赖行为证据 |
| **PDF 电子文件（原生）** | 图层分析、元数据检查、数字签名验证 | — |
| **传真件** | 几乎只能做使用逻辑和内容判断 | 无检验价值 |

### 调查原则

调查用到的系统内数据遵循 **双轨获取** 原则：

- **主路径（primary）**：通过系统接口或 SQL 脚本自动获取，结果可重复验证
- **备用路径（fallback）**：主路径不可用时，由调查员从系统手动提取，结果须人工核验
- **不可获取时**：标注数据缺口（evidence_gap），评估对 finding confidence 的影响

详细的通用调查原则（静默边界、缺口归因、信源前缀）参见 [`skills/investigation-foundation/SKILL.md`](../investigation-foundation/SKILL.md)。

## 分析辅助工具

以下工具类型可辅助加速伪造印章调查。**它们不是必须的**——未配置时，由模型按上述方法直接完成分析。

### 数据库查询类 MCP（如配置）

- **能力：** 执行 SQL 查询、导出电子用印系统日志、印章领用记录
- **辅助场景：** 调取用印记录、提取电子印章数字证书信息、追溯文件创建者
- **不可用时：** 手动查阅用印登记簿、文件元数据查看软件

### 网络搜索类 MCP（如配置）

- **能力：** 公开信息检索、公章备案查询
- **辅助场景：** 公安公章备案系统查询印章编号、查询公章鉴定机构、收集同类案例
- **不可用时：** 联系公安机关或鉴定机构电话查询

### 推理验证类 MCP（如配置）

- **能力：** 分步逻辑推演、多路径比对
- **辅助场景：** 文件流转路径推演、伪造目的分析、盗用时间窗口识别
- **不可用时：** 由模型自行完成推演，输出推演过程供审查
