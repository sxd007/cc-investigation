# 伪造印章调查 — 个人工作手册

面向：正在使用 fraud-fake-chop 技能处理印章真伪问题的调查员。
目的：自己看的操作指南。理解伪造印章调查的两个核心场景、掌握图像鉴定脚本的使用方法、学会结合技术分析与行为证据形成完整证据链。

***

## 一、设计哲学：印章鉴定不是"看图说话"，是"证据拼图"

伪造印章调查（fraud-fake-chop）是一个容易被低估的技能——它不是简单地"对比两个章像不像"，而是一套完整的调查方法论。核心设计围绕三个理念：

**理念 1：图像比对是起点，不是终点。**

`seal_verify.py` 脚本能告诉你"章看起来像不像"，但无法告诉你"盖章行为是否经过授权"。图像比对只能排除或提示伪造，最终结论需要行为证据（用印记录、审批链、访谈）来支撑。**不要把脚本输出当作最终结论。**

**理念 2：调查路径取决于"你手里有什么"。**

调查不是从"这属于哪种伪造形态"开始，而是从"我现在有什么证据"开始：

- **有实物印章线索 → 走实物印章调查路径**（找章、取证、鉴定、溯源）
- **只有文件 → 走业务流程与文件检验路径**（用印记录验证、印文真伪辨别）

**理念 3：工具是你的助手，不是替代你的人。**

脚本做量化分析、AI 做语义判断，但最终决定"要不要委托专业鉴定"的是你。工具提供证据，你做判断。

## 二、理解两个调查场景，选择正确的调查路径

### 2.1 场景速查

```
┌─────────────────────────────────────────────────────────────────────────┐
│  场景一：实物印章调查                                                     │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ 有明确线索指向一枚实物印章是伪造的                                   │  │
│  │ 核心目标：找到物理印章、取证封存、鉴定真伪、溯源非法刻制渠道           │  │
│  │ 适用条件：有实物线索、需要扣押封存印章                               │  │
│  │ 调查路径：寻找印章 → 取证封存 → 真伪鉴定 → 溯源追查 → 够罪评价        │  │
│  └───────────────────────────────────────────────────────────────────┘  │
├─────────────────────────────────────────────────────────────────────────┤
│  场景二：业务流程与文件检验                                               │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │ 只有文件（复印件/扫描件/PDF），没有实物印章                         │  │
│  │ 核心目标：验证用印记录合法性、鉴别文件上印文的真伪                     │  │
│  │ 适用条件：只有文件证据，需判断盖章行为是否经过授权                   │  │
│  │ 调查路径：用印记录验证 → 文件形式鉴别 → 印文真伪辨别 → 行为证据收集  │  │
│  └───────────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 四种伪造形态的检测适用性

在业务流程与文件检验场景中，四种伪造形态的检测策略不同：

| 形态 | 核心特征 | 脚本能否检测 | 关键行动 |
|------|---------|------------|---------|
| 私刻（全假） | 印文本身是仿制的 | ✅ 印文比对（SSIM 低、差异像素高） | 图像比对 + 公安备案查询 |
| 变造（局部改） | 真实印章基础上局部修改 | ⚠️ 噪点一致性可发现局部修改区域 | 原件检验 + 骑缝章比对 |
| 盗用（真章假用） | 章是真的，行为是假的 | ❌ 图像比对无意义 | 用印记录 + 审批链核查 |
| 冒用（PS/合成） | 文件是合成的，印文是贴入的 | ✅ 五项文件真实性检测全覆盖 | 文件真实性检测 + 来源追溯 |

### 2.3 你最容易犯的三个判断错误

1. **看到章不一样就断定是私刻** — 先检查是否有多版本印章（公司可能有新旧版章），再查公安备案。
2. **看到章一样就认为没问题** — 盗用形态就是真章假用，图像比对通过不等于盖章行为合法。
3. **只有扫描件就放弃分析** — 扫描件至少能做文件真实性检测（ELA、图层结构），结合行为证据仍可形成结论。

## 三、场景一：实物印章调查操作指南

### 3.1 寻找实物印章

| 场景 | 搜索策略 |
|------|---------|
| 有明确存放线索 | 在指定位置（办公室抽屉、文件柜、保险柜）搜索；建议有见证人在场 |
| 有使用人线索 | 跟踪印章使用记录，定位最后使用人和存放位置 |
| 只有举报线索 | 从用印记录和审批链反推印章可能存放的位置，结合监控和门禁记录缩小范围 |

**注意事项：**
- 搜索时应有**至少两人在场**（主调查员 + 见证人），全程记录
- 发现疑似伪造印章后，**先拍照固定位置**，再用干净手套取出
- 不要直接触碰印章面（防止污染指纹证据或破坏油墨）

### 3.2 取证封存

物理印章的取证封存遵循链式保管原则：

```
发现印章 → 原位拍照 → 戴手套取出 → 装入证物袋（标注发现时间/地点/发现人）
→ 封口处由发现人和见证人双签 → 移交证物室（记录移交链）

封存要点：
• 证物袋上标注：案件编号、证物编号、提取人、提取时间、地点
• 印章面朝上，不与袋壁摩擦（防止印文模糊）
• 如印章上有油墨未干，先自然晾干再封存
• 电子印章（U盾类）单独封存，标注"电子介质"
```

### 3.3 真伪鉴定

**调查员层（初步判断）：**

| 检查项 | 方法 | 判断依据 |
|--------|------|---------|
| 材质与工艺 | 目视+手感 | 真章通常是铜/钢制刻制，私刻常用廉价金属或光敏/原子印 |
| 字体与排版 | 与存档印模叠图比对 | 字体差异（仿宋vs宋体）、文字间距、缺笔少划 |
| 尺寸 | 精确测量直径 | 与公安备案尺寸相差1mm以上可疑 |
| 编号 | 查询公安公章备案系统 | 编号不存在或属于其他单位 |
| 做旧痕迹 | 放大镜观察磨损分布 | 人为做旧磨损均匀，自然磨损不均匀（边框比中心磨损重） |
| 刻制工艺 | 放大镜观察笔画边缘 | 真章刻制笔画干净，私刻边缘有毛刺或抖动 |

**鉴定机构层（专业鉴定）：**
- 委托司法鉴定机构进行印文同一性鉴定
- 委托电子数据鉴定进行电子印章真伪核验
- 调查员负责提供比对样本（公安备案印鉴卡、公证章模、多份确认真实的文件）

### 3.4 溯源调查

找到伪造印章后，需追溯其来源以形成完整证据链：

| 溯源方向 | 调查方法 | 可能的证据 |
|---------|---------|-----------|
| **刻制来源** | 询问印章使用人"在哪里刻的"；搜索其通信记录（微信/钉钉）中"刻章""印章"等关键词 | 刻章店转账记录、聊天记录、收据 |
| **网络购买** | 检查当事人网购平台（淘宝/京东/拼多多）购买记录 | 订单截图、物流记录 |
| **刻章店实地** | 如来源明确，实地走访刻章店，调取监控和营业记录 | 刻章店笔录、监控截图、刻章登记 |
| **刻制时间推定** | 从用印记录中推定首次使用该假章的时间 | 用印记录、文件日期 |

### 3.5 够罪评价

《刑法》第 280 条第 2 款：**伪造公司、企业、事业单位、人民团体印章的，处三年以下有期徒刑、拘役、管制或者剥夺政治权利，并处罚金。**

| 评价维度 | 考量因素 |
|---------|---------|
| **构成要件** | 是否"伪造"（无权制作而仿制）即构成，不要求实际使用 |
| **数量情节** | 伪造枚数、使用频次 |
| **后果情节** | 是否造成经济损失、是否用于其他犯罪（合同诈骗等） |
| **主观故意** | 是否明知是伪造而使用 |
| **实务要点** | 该罪是行为犯——只要有伪造行为即构成，不要求实际造成损失 |

> ⚠️ 够罪评价仅供调查员内部参考，最终定性须由司法机关依法认定。调查员的任务是收集证据，不替代司法判断。

## 四、场景二：业务流程与文件检验操作指南

### 4.1 用印记录验证

验证文件上的印章使用行为是否真实经过了审批流程：

| 检查项 | 查验内容 | 异常信号 |
|--------|---------|---------|
| **审批链完整性** | 文件类型是否需要审批、审批节点是否齐全 | 关键审批节点缺失、审批人否认签署 |
| **时间合理性** | 用印申请时间 vs 文件盖章时间 vs 保管人出勤时间 | 章盖在保管人休假期间、审批在前盖章在后间隔异常 |
| **地点合理性** | 电子用印系统的 IP/设备位置 vs 印章存放地点 | 同一印章同一天出现在两个城市 |
| **用途匹配** | 用印申请单填写的用途 vs 实际文件内容 | 申请"资质文件"实际盖在合同上 |
| **频次异常** | 同一印章在短时间内被高频调用 | 异常密集使用（暗示批量伪造） |

**数据来源：**
- 电子用印系统操作日志（最佳）
- 纸质用印登记簿（次选，需防篡改检查）
- 保管人访谈（补充）

### 4.2 文件形式鉴别

| 文件形式 | 可做的判断 | 不可做的判断 |
|---------|-----------|-------------|
| **纸质原件** | 物理特征检验、叠图比对、油墨分布、压痕深度 | 无法独立判断电子印章真伪 |
| **高分辨率扫描件（600dpi+）** | 尺寸比例、字体细节、印章+文字关系、PDF 元数据 | 无法做物理特征检验（油墨、纸张） |
| **低分辨率扫描/拍照件（<300dpi）** | 宏观特征（印文位置、内容、使用逻辑） | 无法做细部比对，依赖行为证据 |
| **PDF 电子文件（原生）** | 图层分析、元数据检查、数字签名验证、`seal_verify.py` | — |
| **传真件** | 几乎只能做使用逻辑和内容判断 | 无检验价值 |

### 4.3 印文真伪辨别

**肉眼/放大镜观察（有纸质原件时）：**

| 观察项 | 真实印章特征 | 伪造特征 |
|--------|------------|---------|
| 印文边缘 | 油墨自然扩散，边缘微模糊 | 过于锐利（数字打印），或有规律性网点（印刷） |
| 字体笔画 | 均匀一致 | 缺笔少划、粗细不均、间距不自然 |
| 油墨分布 | 中间浓边缘淡，有压痕 | 均匀平坦（打印），或 RGB 合成色 |
| 印章+文字关系 | 印章盖在文字之上（印文遮盖笔画） | PS 合成时印章可能在文字层之下 |
| 纸张压痕 | 背面有凹凸压痕 | 无压痕 |

**脚本辅助检验（有扫描件/PDF 时）：**
调用 `seal_verify.py` 进行量化分析 → [见第五章](#五、用好图像鉴定脚本)

### 4.4 行为证据收集

印文检验只能回答"章是不是真的"——要回答"章是谁盖的、是否获得了授权"，需要行为证据：

- **访谈印章保管人：** 盖章当天的具体情况、是否有异常人员接触、是否有异常文件
- **访谈审批人：** 是否本人签署审批单、是否知晓该文件内容
- **追溯文件流转路径：** 文件是谁制作的、通过什么渠道传递的（邮件/IM/系统上传）
- **交易对手核实：** 对方是否收到文件、盖章窗口期是否合理
- **监控与门禁记录：** 印章存放地点在盖章时间段的出入记录

## 五、用好图像鉴定脚本

### 5.1 脚本能做什么

`seal_verify.py` 是你最核心的技术工具。它执行两轨分析：

**第一轨：印章形态比对**

- SSIM 结构相似度（衡量整体结构一致性）
- 红色区域 IoU（衡量红色印章区域的重合度）
- 差异像素比例（衡量两章的像素级差异）
- 叠加对比图 + 差异热图（可视化差异位置）

**第二轨：文件真实性检测（新增 v2）**

- **ELA 错误级别分析**：检测 JPEG 压缩历史差异，识别 PS 粘贴痕迹
- **PDF 元数据分析**：检查创建工具、修改时间、编辑软件痕迹
- **PDF 图层结构分析**：检查独立图像对象数量，识别合成文件
- **噪点一致性分析**：对比印章与背景的噪点模式，判断来源一致性
- **边缘锐利度分析**：区分数字打印（锐利）与真实盖章（自然模糊）

### 5.2 脚本不能做什么

| 脚本不能做              | 正确做法       |
| ------------------ | ---------- |
| 判断盖章行为是否经过授权       | 调取用印记录、审批链 |
| 鉴定电子印章的加密签名        | 委托电子数据鉴定机构 |
| 做物理特征检验（油墨成分、纸张纤维） | 委托司法鉴定机构   |
| 给出具有法律效力的鉴定意见      | 脚本结论仅供调查参考 |

### 5.3 调用方式

```bash
# 基本用法：对比检材和样本
python scripts/seal_verify.py <检材文件> <样本文件>

# 指定输出路径
python scripts/seal_verify.py suspect.pdf original.png --output report.html
```

**参数说明：**

| 参数       | 说明          | 支持格式                 |
| -------- | ----------- | -------------------- |
| 检材文件     | 待鉴定的可疑文件    | jpg, png, pdf        |
| 样本文件     | 真实印模参考文件    | jpg, png, pdf        |
| --output | HTML 报告输出路径 | 默认：seal\_report.html |

**输入要求：**

- PDF 文件自动提取第一页进行分析
- 图像分辨率建议 ≥ 300dpi
- 印章区域需清晰可见

### 5.4 输出解读

**命令行摘要（Agent 消费）：**

```
✅ 印章比对：真实（高度一致）
📊 综合评分：92.3%
✅ 文件真实性：无明显PS/抠图痕迹
📄 报告：seal_report.html
```

**报告内容（人工审查）：**

| 区块      | 内容                | 用途        |
| ------- | ----------------- | --------- |
| 印章对比图   | 检材章、样本章并排展示       | 目视初步判断    |
| 量化指标卡片  | SSIM、IoU、差异像素比    | 量化评估      |
| 叠加差异图   | 红蓝叠加 + 差异轮廓标注     | 精确定位差异位置  |
| 差异热图    | 蓝→绿→红表示差异程度       | 可视化差异分布   |
| 文件真实性检测 | 5 项检测的状态、详细分析、可视化 | 判断是否有后期处理 |
| AI 专家分析 | 语义层面的专业意见（可选）     | 辅助判断      |

### 5.5 结论分级解读

| 综合评分    | 结论     | 含义          | 后续行动        |
| ------- | ------ | ----------- | ----------- |
| ≥ 85%   | ✅ 真实   | 高度一致，各项指标正常 | 采信，结合行为证据验证 |
| 70%-85% | ⚠️ 存疑  | 部分差异，需人工判断  | 调取原件或委托鉴定   |
| < 70%   | ❌ 疑似伪造 | 差异显著，伪造可能性高 | 启动正式鉴定流程    |

**文件真实性检测的额外判断：**

- 只要有一项检测标记为"可疑"，即使印章比对通过，也要警惕
- ELA 可疑 → 高度提示印章为后期粘贴
- 图层结构可疑 → 高度提示文件为合成

**AI 专家分析（可选）：**

```bash
python scripts/seal_verify.py ...
```

## 六、双轨取证策略

伪造印章调查需要同时推进三条证据线：

```
证据线 A：印章本体证据
  ├── 可疑印章文件（检材）
  ├── 真实印模样本
  ├── 公安备案记录
  ├── 脚本分析报告
  └── （如必要）专业鉴定意见
  → 证明"章本身是假的"

证据线 B：行为证据
  ├── 用印记录 / 电子用印系统日志
  ├── 审批链 / 用印审批单
  ├── 保管人访谈笔录
  ├── 文件流转路径
  └── 交易对手证言
  → 证明"使用印章的行为是未经授权的"

证据线 C：目的证据
  ├── 文件签署背景
  ├── 资金流向
  └── 相关交易记录
  → 证明"伪造印章的目的"
```

**三条证据线互相补充。** 即使证据线 A 无法得出明确结论（如只有扫描件），证据线 B + C 也可能形成完整证据链。反之，鉴定了印章造假但无法证明使用目的，也需要行为证据补强。

## 七、你与 Agent 的协作模式

### 7.1 标准协作流程

```
1. 你：描述可疑印章文件的情况
   → 系统自动识别是否激活 fraud-fake-chop 技能

2. 你：判断调查场景
   → 有实物线索 → 走场景一（实物印章调查）
   → 只有文件 → 走场景二（业务流程与文件检验）

3. 你：调用图像鉴定脚本（如适用）
   → 上传检材和样本文件
   → seal_verify.py 生成分析报告

4. 你：解读报告
   → 查看印章比对结论
   → 查看文件真实性检测结果
   → 判断是否有可疑项

5. 你：补充行为证据
   → 调用 data-analyzer 调取用印记录
   → 调用 evidence-analyzer 评估证据链

6. 你：形成 finding
   → 将分析结果写入 EV 证据节点
   → 标注置信度和支撑证据
```

### 7.2 常见场景的处理方式

**场景 1：对方只提供扫描件，拒绝提供原件**

```
操作：
  1. 调用脚本做文件真实性检测（重点看 ELA、图层结构）
  2. 调取用印记录核查是否有对应的盖章记录
  3. 访谈经办人了解文件来源
  4. 标记"证据降级"，评估对 finding confidence 的影响
```

**场景 2：印章比对通过，但文件真实性检测可疑**

```
操作：
  1. 这是典型的"盗用"或"冒用"形态
  2. 重点调查用印记录和审批链
  3. 核查盖章时间是否合理（非工作时间？异地使用？）
  4. 追溯文件电子版的来源路径
```

**场景 3：印章比对存疑（70%-85%）**

```
操作：
  1. 调取纸质原件做物理检验
  2. 测量印文精确尺寸（直径、文字间距）
  3. 检查印章编号与公安备案是否一致
  4. 比对多份已确认真实文件上的印章，确认是否有多版本差异
```

**场景 4：发现疑似伪造印章实物**

```
操作：
  1. 原位拍照固定证据位置
  2. 戴手套取出，装入证物袋
  3. 双人签字封存，建立保管链
  4. 委托专业鉴定机构进行印文同一性鉴定
  5. 追溯印章来源（刻章店、网络购买等）
```
