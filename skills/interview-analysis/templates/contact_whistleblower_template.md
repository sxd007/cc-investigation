# 举报人联系模板

> **使用前提醒：**
> 1. **讨论推敲后再决策** — 本模板是参考框架，不是执行脚本。每个问题、每句话术都应根据案件具体情况、掌握的证据、举报人的背景和状态进行讨论后调整。
> 2. **背景核查先于联系** — 在发出任何联系前，确认已通过手机号/邮箱完成举报人的背景核查（公司关联、司法风险）。
> 3. **信息对等原则** — 在获取可核查的核心信息（公司名称、具体证据、合同文件）之前，不披露调查方法论、法律处置路径、报案策略等操作细节。
> 4. **一次一复盘** — 每次联系后必须完成评估，再决定是否/如何进行下一次联系。

## 模板A：电话回访提纲

### A0. 通话前检查清单

复制以下内容到案件工作目录，逐项确认后开始通话：

```
□ 举报人背景核查已完成
   - 关联公司：__________（如有负面信息：__________）
   - 司法风险：__________
   - 本次通话策略调整：__________
□ 举报人联系方式确认：__________
□ 本次通话目标（选择主要目标）：
   ○ 获取被举报公司全名
   ○ 获取具体证据材料（照片/合同/记录）
   ○ 核实举报人身份和动机
   ○ 其他：__________
□ 信息交换底线：在获取__________之前，不讨论__________
□ 录音准备：本次通话是否同意对方录音？ □ 是（策略：__________） □ 否
□ 记录人/记录方式：__________
```

### A1. 开场

**首次通话：**
> "您好，[举报人称呼]，我是 [公司名] 合规调查团队的 [姓名]。您 [日期] 发给我们的举报邮件我们已经收到了，非常感谢您愿意反映情况。今天想跟您了解一些具体情况。通话大约需要 [预估时长] 分钟。"

**后续通话（已有过接触）：**
> "[举报人称呼]，我是 [公司名] 的 [姓名]。上次通话后我们做了一些内部工作，今天想再跟您确认几个细节，方便吗？"

**举报人投诉热线打不通时增加道歉：**
> "也要跟您说声抱歉——您之前反映的热线打不通的问题，我们已经上报处理了。以后您可以直接联系我。"

### A2. 举报人身份核实（首次通话必问）

```
Q1: "能否简单介绍一下您自己？您目前在哪个公司、负责什么业务？"
    追问：____________________

Q2: "方便问一下，您是怎么了解到我们的举报渠道的？"
    追问：____________________

Q3: "您和我们公司的业务有直接合作吗？大概多久了？"
    追问：____________________

Q4: "您希望通过这次反映达到什么目的？"
    观察：□ 利益受损者 □ 同行竞争 □ 维护秩序 □ 其他：__________
```

### A3. 被举报方信息核实（核心问题）

```
Q5: "您邮件里提到的 [公司/个人]，它的全名/全称是什么？"
    ⚠️ 如果对方回避，标记为警示信号
    追问：____________________

Q6: "您和这家公司/这个人有过直接接触吗？是什么样的接触？"
    追问：____________________

Q7: "您方便提供他们的联系人或相关信息吗？"
    追问：____________________
```

### A4. 举报核心事实深挖（根据举报内容选择对应模块）

**模块：刮码/改码行为**
```
Q: "您提到的改码——能具体说说他们是怎么做的吗？"
   追问方向：改了什么码？在哪里改的？您是怎么发现的？

Q: "您手里有没有相关的照片、视频或实物样品？"
   追问方向：拍摄时间、地点、目前保存在哪？
```

**模块：假合同/假项目**
```
Q: "假合同/假项目的事——能具体讲讲是什么情况？"
   追问方向：冒用了谁的名义？您见过文件吗？如何获取的？

Q: "这份材料您方便提供给我们吗？"
   ⚠️ 如果对方犹豫，不要追问来源，可以先以匿名方式接收
```

**模块：窜货/违规销售**
```
Q: "您观察到货物流向了哪些区域？"
   追问方向：运输方式、物流单据、时间频率

Q: "这个情况持续了多久？您最早是什么时候发现的？"
   追问方向：时间范围、批次数量
```

### A5. 举报人常见反问及参考回应

**Q: "你们会怎么处理他/他们？"**
> 参考回应："我们还在了解情况阶段，等掌握具体信息后才能评估。您如果能提供 [公司名称/证据材料]，我们才能推进下一步。"

**Q: "你们调查会不会被内部人干扰？"**
> 参考回应："我们有独立的调查流程和管控机制。不过您能告诉我们更多具体线索，我们核实起来会更准确。"

**Q: "你们处理完能不能让 [某个机构/某人] 来办？"**
> 参考回应："具体由哪个部门处理，需要根据调查结果和涉及的行为性质来定。我们现阶段先把事实弄清楚。"

**Q: "能不能给我一些报酬？"**
> 参考回应："公司对有效举报有奖励机制，但需要等调查结论出来后才能评估。我们先集中精力把问题核实清楚。"

### A6. 收尾

```
"非常感谢您今天提供的信息。我会把情况做内部记录。

接下来：
1. 我们会进行内部核查
2. 有进展我会主动联系您

如果之后还想起什么，可以随时打这个电话，或发邮件到 [联系方式]。"

□ 确认了后续联系方式和联系频率
□ 明确了信息的保密处理方式
□ 通话时长控制：____分钟（是否符合预期）
```

## 模板B：邮件回复与追问

### B0. 发件前检查清单

```
□ 举报人背景核查已完成
   - 关联公司：__________
   - 司法风险：__________
□ 本次邮件目标：
   ○ 回复初次举报（致谢 + 提问）
   ○ 追问补充信息（针对已有线索）
   ○ 回应举报人提出的问题
   ○ 其他：__________
□ 信息交换底线：__________
□ 附件要求：__________
□ 抄送/密送：□ 需要 □ 不需要（按合规流程决定）
```

### B1. 初次回复邮件模板

**主题：** 关于您 [日期] 提交的合规举报 — [案件编号/内部参考号]

**正文：**

> [举报人称呼] 您好，
>
> 感谢您 [日期] 通过邮件向我们反映情况。我们已对您提供的信息进行了初步登记（内部编号：[案件编号]）。
>
> 为了能够进一步核实您提到的情况，想请您补充以下信息：
>
> 1. **关于被举报方**：您提到的 [公司/个人] 的全称是什么？您是否了解其与我们公司的业务关系？
> 2. **关于您掌握的信息**：您提到的 [具体问题] 是基于您个人的直接了解，还是通过他人获知的？
> 3. **关于证据材料**：您是否持有相关的合同、照片、交易记录或聊天记录等材料？如有，是否可以提供给我们？
>
> 您可以通过回复本邮件或致电 [调查员电话] 与我们联系。
>
> 再次感谢您对合规工作的支持。
>
> 此致
>
> [姓名]
> [公司名] 合规调查团队
> [联系电话]
> [邮箱]

### B2. 追问/跟进邮件模板

**主题：** 关于 [案件编号] 的后续补充了解

**正文：**

> [举报人称呼] 您好，
>
> 感谢您上次与我们通话/邮件沟通。我们就您提供的信息进行了一些基础核查，目前还有几个关键信息需要向您确认：
>
> 1. [具体问题1，如：您提到的项目是通过哪家代理商申报的？]
> 2. [具体问题2，如：涉及的物资具体是什么型号？大概数量是多少？]
> 3. [具体问题3，如：您提到的照片是否方便通过邮件附件发送给我们？]
>
> 以上信息对我们判断情况非常有帮助。如您有任何顾虑，我们可以进一步沟通如何处理这些材料。
>
> 期待您的回复。
>
> 此致
>
> [姓名]
> [公司名] 合规调查团队

### B3. 举报人反问的邮件回应

**场景：举报人询问处置方案**

> 关于您提出的问题，我们的调查目前处于信息收集阶段，具体的处置方案需要在查清事实后才能确定。当前最关键的，是请您协助提供基础信息（如被举报方的全称、相关材料等），以便我们推进工作。

**场景：举报人询问是否会通知被举报方**

> 我们在调查期间会严格保密信息来源。未经必要程序，不会向被举报方透露举报人的任何信息。

## 通话/邮件后行动清单

通话或邮件发出后，逐项完成：

```
□ 通话/邮件记录归档
   - 文件命名：03_call_memo_YYYYMMDD.md 或 03_email_copy_YYYYMMDD.md
   - 核心内容：记录了时间、参与人、关键问答、承诺事项

□ 信息缺口表更新（IG-xx）
   - 已填补：__________
   - 新增缺口：__________

□ 举报人可信度评估更新
   - 本次联系后评估：□ 提升 □ 不变 □ 下降
   - 原因：__________

□ 警示信号检查
   - □ 无异常
   - □ 存在以下信号，需暂停评估：
     1. __________
     2. __________

□ 下一次联系计划
   - □ 需要再次联系，目标：__________
   - □ 不需要再次联系，原因：__________
   - □ 需暂停并内部讨论后再决定

□ 证据登记（如获取了新材料）
   - evidence_registry.json 已更新：□ 是 □ 否
```
