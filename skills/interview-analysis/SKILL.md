# 访谈与问话分析

调查访谈是获取信息最重要的手段，也是最考验调查员功力的环节。本技能覆盖访谈全流程：**访谈前的策略设计与提纲生成、访谈后的笔录结构化审查与对抗行为识别**，并提供 SCAN 陈述分析和 PEACE 模型作为方法论参考。

## 配置前置检查

在执行本技能的业务操作前，按以下流程检查用户配置：

```
检查 ~/.claude/plugins/config/cc-investigation/team-profile.md
├── 不存在 / 含 [PLACEHOLDER] / 含 PAUSED 标记
│   └── 停止操作，提示: "请先运行 /cc-investigation:cold-start-interview 完成设置"
└── 配置就绪 → 继续
```

详细规则参见 `config-templates/config-loader.md`。

此技能读取的配置项：
- team-profile → 调查通信纪律：通话上限、背景核查要求、call_memo 强制输出（待后续实现行为约束）

## When to Activate

- 策划和准备调查访谈、设计问话策略
- 生成访谈提纲或评估已有提纲的完整性
- 分析已完成的谈话笔录，输出结构化分析报告
- 识别受访者的虚假配合、选择性配合、编造性拒绝、经验性回避等对抗行为
- 评估笔录的清晰度、连贯性和证明力
- 评估访谈人的提问有效性和追问质量
- 多轮访谈的一致性比对
- 生成下一轮追问建议和独立取证任务
- 评估证言可信度（SCAN 方法）
- 培训访谈技巧

## 一、访谈策略与提纲设计

### 1.1 对象分类与问话策略

```
被访谈人类型 ─┬── 证人 (Witness)        → 信息采集者心态
              ├── 被调查人 (Subject)     → 核实证据为主
              ├── 举报人 (Whistleblower)  → 验证可信度
              └── 专家 (Expert)          → 获取专业意见
```

| 维度 | 证人 | 被调查人 | 举报人 |
|------|------|---------|--------|
| **氛围** | 合作、轻松 | 有压力、权威 | 安全、保护 |
| **开场** | 你观察到什么？ | 我们需要你帮助澄清一些事 | 感谢你站出来 |
| **信息获取** | 开放式问题为主 | 闭合式+渐进揭露证据 | 细节深挖 |
| **核实** | 交叉验证其他证人 | 用证据戳破矛盾 | 独立证据验证 |
| **陷阱** | 记忆偏差 | 撒谎/隐瞒 | 动机不纯/夸大 |

### 1.2 提纲生成标准

生成访谈提纲时，必须包含以下模块。**每个问题必须是完整可用的问句措辞，不得只给出追问方向。**

**提纲结构：**

```
一、开场陈述
   · 自我介绍与角色说明
   · 访谈目的与范围说明
   · 保密声明/权利告知（按适用场景）
   · 基本规则（如实回答、不确定时说不知道）

二、背景信息采集
   · 受访者岗位/职责/任职时间
   · 与案件/事项的关联
   · 相关流程的常规操作基线

三、核心问题序列
   · 按时间线或主题分层组织
   · 从开放 → 聚焦 → 闭合的递进结构
   · 每个主题区至少预设 1-2 条追问路径

四、核实与交叉验证
   · 针对已有证据设计的核实性提问
   · 对矛盾点的测试性提问
   · 对缺失信息的覆盖性提问

五、收尾
   · 是否有补充？
   · 是否可以后续联系？
   · 签字确认安排
```

**对象类型对提纲的影响：**

| 设计要点 | 证人 | 被调查人 | 举报人 |
|---------|------|---------|--------|
| 信息获取方式 | 开放式为主 | 渐进揭露为主 | 深挖式追问 |
| 证据使用策略 | 无需展示 | 选择性揭露 | 验证性展示 |
| 对抗行为预判 | 低 | 高 | 中（注意夸大） |
| 问题密度 | 低→中 | 高 | 中 |

### 1.3 提纲评估标准

使用以下清单评估一份访谈提纲是否完整有效：

**结构完整性：**
- [ ] 是否包含开场/背景/核心/核实/收尾五个模块？
- [ ] 核心问题是否按时间线或主题分层？
- [ ] 是否预设了追问路径（对关键缺口至少 1-2 条）？

**策略适配性：**
- [ ] 问题类型是否适配受访者类型（证人→开放、被调查人→渐进）？
- [ ] 是否设计了证据使用策略（何时展示、展示什么）？
- [ ] 是否有对抗行为的预判和应对预案？

**可执行性：**
- [ ] 每个核心问题是否可被"直接朗读使用"？
- [ ] 是否明确了每类问题的预期信息产出？
- [ ] 是否标注了优先级（哪些问题必须问到）？

### 1.4 PEACE 模型（参考）

源自英国警方审讯模型，以下仅作方法论参考，不强制遵循：

| 阶段 | 内容 |
|------|------|
| **P**lanning & Preparation | 计划和准备 |
| **E**ngage & Explain | 建立关系、说明目的 |
| **A**ccount | 获取陈述（自由叙述 → 追问） |
| **C**losure | 结束、总结、确认 |
| **E**valuate | 评估、与证据对比 |

## 二、笔录结构化审查

对已完成谈话笔录进行系统性结构化分析，输出可直接用于下一轮谈话准备和独立取证安排的分析报告。详细评级标准和对抗行为识别框架见 `transcript-review/references/interview_review_rubric.md`。

### 2.1 适用场景与输入

**必填：**
- 本轮谈话笔录全文
- 当前核心假设列表（每条一句话）

**选填（多轮迭代时提供）：**
- 上一轮笔录分析报告
- 上一轮追问清单和取证任务

选填项未提供时，跳过多轮一致性比对，仅输出单轮分析结果。

**前置动作：** 读取 `transcript-review/references/interview_review_rubric.md` 获取完整评级标准和识别框架。读取 `transcript-review/assets/report_template.html` 作为输出模板。

### 2.2 受访者身份判断

分析前先判断受访者身份，身份类型影响对抗行为的识别逻辑和陈述性质的权重：

| 受访者类型 | 编造性拒绝是否适用 | 陈述性质权重 |
|-----------|----------------|------------|
| 内部人员（在职员工、管理层） | 适用 | 亲历陈述预期最高，缺位本身可疑 |
| 关联方（离职员工、派驻人员） | 部分适用，需结合配合义务判断 | 预期部分亲历+转述混合 |
| 外部第三方（供应商、客户、证人） | 不适用，无配合义务 | 转述/推测比例高属正常 |

### 2.3 分析流程

按顺序执行，不得跳步。

---

**Step 1：提取基本信息与陈述性质分类**

从笔录中识别：受访者姓名和职务、身份类型、谈话时间、本轮核心议题。

同时对每个问答块的陈述进行性质分类，标注在每个问答块上：

| 性质 | 定义 | 文本信号 | 证据价值 |
|------|------|---------|---------|
| **firsthand** 亲历 | 陈述人直接经历或操作 | "我看到了""我经手的""我在现场" | 最高 |
| **relay** 转述 | 陈述人转述他人信息 | "XX告诉我的""听他们说" | 次之，需核实来源 |
| **opinion** 推测 | 陈述人的判断或推测 | "我觉得""可能""应该是" | 低 |
| **attitude** 态度 | 态度表达，无事实陈述 | "我们一直很规范""不可能" | 无独立证据价值 |

分类原则：优先按最高可信度归类——同一问答块含亲历和推测成分的，归为首要性质的类别，但 marks 中标注切换点。

---

**Step 2：笔录质量评级**

依据 `transcript-review/references/interview_review_rubric.md § 二` 中的标准，对以下 **四个维度** 分别评级：

**① 清晰度**：逐段扫描笔录，将所有模糊表述穷举列出——包括不明指代、模糊时间、抽象动词、无法追索的材料描述。不得只举 1-2 个例子，必须列出笔录中全部此类问题实例，并注明出现在哪个问答环节。

**② 连贯性**：从笔录中还原完整事件链（起因→过程→决策→执行→结果），逐一标出每个跳步和解释空洞的位置，说明：(1) 缺失的是哪段链条，(2) 对应的是哪个问答环节，(3) 访谈人在该环节是否未能有效追问。不得只给总体判断，必须逐条列出所有结构性断裂。

**③ 证明力**：不做孤立评级，必须锚定在每条核心假设上分别说明：(1) 本轮笔录对该假设已提供了哪些可验证的支撑，(2) 要达到"高"证明力还缺少哪些具体材料或信息，(3) 当前最薄弱的证明环节在哪里。结合 Step 1 的陈述性质分类，对 firsthand/relay/opinion/attitude 四类陈述分别说明各自贡献了多少证明力。

**④ 追问质量（新增）**：评估访谈人在本轮谈话中的提问表现：

| 评级 | 判断标准 | 典型问题信号 |
|-----|---------|------------|
| 高 | 关键缺口穷尽追问，提问节奏合理，未被受访者带偏 | 清晰的问题递进链；受访者转移话题后被拉回 |
| 中 | 主要缺口有追问但不够深入，或部分明显缺口被放过 | 受访者给出抽象回答后未追问具体细节 |
| 低 | 大量关键缺口未追问，访谈人被受访者主导节奏 | 受访者回避后访谈人直接放弃或转移话题 |

追问质量的 marks 与连贯性 gap 联动——一个 gap 的出现，可能同时是"受访者回避"和"访谈人未追问"的结果，应同时标注。

---

**Step 3：对抗行为识别**

依据 `transcript-review/references/interview_review_rubric.md § 三` 中的识别框架，逐一检查四类对抗行为：

- **虚假配合**：声称已配合但描述存在关键缺失、来源不明或与其他证据冲突
- **选择性配合**：对外围瑕疵快速承认，对关键链路持续模糊
- **编造性拒绝**：以保密/权限等为由拒绝，但缺少具体依据（仅适用于内部人员及部分关联方）
- **经验性回避**：亲历者对关键细节无法说明，使用"惯例如此""记不清了"

外部第三方跳过编造性拒绝的检查。

对抗行为识别须结合 **追问质量评估**：对抗行为的严重程度不仅取决于受访者的表现，也取决于访谈人是否给予了有效的追问机会。受访者在一个未被追问的缺口上保持沉默，与在一个穷尽追问的缺口上保持沉默，意义不同。

---

**Step 4：假设影响判断**

逐条对照输入的核心假设，判断本轮笔录对每条假设的影响方向：

- **strengthen**（增强）：该假设获得新的可验证支撑
- **neutral**（暂无影响）：既未增强也未削弱
- **weaken**（削弱）：出现了与该假设矛盾的信息

每条假设必须分别说明：
- `has`：本轮已提供的可验证支撑
- `lacks`：距高证明力仍然缺少的材料或信息

---

**Step 5：多轮一致性比对**（仅在提供上一轮报告时执行）

依据 `transcript-review/references/interview_review_rubric.md § 四` 中的比对框架：
- 陈述一致性检查（一致/差异/矛盾/补充）
- 追问清单覆盖情况（已充分回应/部分回应/未回应/回避回应）
- 对抗行为变化（持续/升级/减弱/消失/新增）

对于多轮访谈，应建立 **陈述演变追踪表**，对每个关键事实记录其在不同轮次的版本。当版本从具体变为模糊/从否认变为承认/从一致变为矛盾时，标记为高优先级疑点。

---

**Step 6：生成后续行动建议**

依据 `transcript-review/references/interview_review_rubric.md § 五` 中的框架：

- **A 类（下一轮谈话追问）**：针对笔录缺口的补充追问、针对对抗行为的针对性追问、针对多轮矛盾的核实追问。每一条必须是可直接使用的问题措辞。
- **B 类（独立取证任务）**：系统侧证、对手侧证、文档侧证、逻辑侧证。每一条必须说明目标材料名称或具体信息来源。

**原则：每识别出一个关键对抗行为，至少同步输出一条 B 类独立取证路径。**

---

### 2.4 输出格式与报告模板

输出为完整的 HTML 报告，严格遵循 `transcript-review/assets/report_template.html` 的结构。

报告包含四个数据填充区：

**`META`** — 基本信息（受访者、身份类型、谈话时间、议题、轮次）

**`BLOCKS`** — 笔录问答块数组，每个块含：
```js
{
  id: 1,
  q: "问题原文",
  a: "回答原文（纯文本，不含标记）",
  nature: "firsthand | relay | opinion | attitude",  // 陈述性质分类
  marks: [
    {
      id: "a1-v1",
      type: "vague | gap | resist | pursuit",  // pursuit = 追问质量问题
      subtype: "vague:pronoun | vague:time | vague:verb | vague:material | gap:skip | gap:contradict | gap:diversion | resist:false_coop | resist:selective | resist:fabricated | resist:evasion | pursuit:missed | pursuit:insufficient | pursuit:lost",
      quote: "被标注的原文片段",
      note: "问题说明",
      env: "出现环节说明 · 访谈人是否有效追问"
    }
    // 同一问答块可有多个 mark，穷举所有问题
  ]
}
```

**`SUMMARY`** — 汇总分析（清晰度/连贯性/追问质量/证明力评级、对抗行为汇总、假设影响矩阵、多轮比对结果）

**`ACTIONS`** — 后续行动建议数组（A 类追问 + B 类取证）

## 三、分析辅助工具

以下工具类型可辅助加速笔录审查分析。**它们不是必须的**——未配置时，由模型按 rubric 标准直接完成所有分析步骤。

### 3.1 语言分析类 MCP（如配置）

- **能力：** 检测文本中的情感模式、模糊限制语分布、第一人称缺失模式
- **辅助场景：**
  - Step 2 清晰度分析：辅助检测模糊表述的分布密度
  - Step 3 对抗行为识别：辅助标记情感突变点（可能对应敏感话题切入处）
  - CAUTIOUS 词库辅助分析：检测去个人化表述、过度修饰语等语言特征
- **不可用时：** 由模型逐段扫描笔录文本，按 rubric 标准直接标记

### 3.2 推理验证类 MCP（如配置）

- **能力：** 分步逻辑推演、多路径一致性验证
- **辅助场景：**
  - Step 4 假设影响判断：辅助对多条假设路径进行交叉推演
  - Step 5 多轮比对：辅助系统化比对跨轮陈述的逻辑一致性
- **不可用时：** 由模型自行完成推演，输出推演过程供审查

### 3.3 文件系统类 MCP（如配置）

- **能力：** 文件读写、目录管理
- **辅助场景：** 将生成的 HTML 报告写入证据底稿目录
- **不可用时：** 在对话中输出报告全文，由用户自行保存

**重要：** 语言分析 MCP 的输出在报告中应标注为"工具辅助提示"，与人工分析结论区分。

## 四、未来扩展

以下方向需独立工具或架构支持，暂不纳入本技能的核心分析流程：

- **实时访谈辅助**：访谈过程中的即时对抗行为标记、话术提示、追问时机提醒。需要低延迟音频处理链路，当前 Claude Code 的推理模式不支持实时交互场景。
- **录音转写比对**：将原始录音转写文本与笔录进行一致性比对，自动标记删减/修改/添加处。需要音频处理 MCP 能力，待相关 MCP 服务器成熟后集成。
- **多语言场景判断**：非母语受访者的模糊表述可能源于语言能力而非对抗行为。需要语言能力评估框架，目前依赖分析者经验判断。
