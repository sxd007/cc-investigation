# 调查工具与技术

一套顺手的调查工具箱 — 不同场景选不同的工具。

## 配置前置检查

在执行本技能的业务操作前，按以下流程检查用户配置：

```
检查 ~/.claude/plugins/config/cc-investigation/team-profile.md
├── 不存在 / 含 [PLACEHOLDER] / 含 PAUSED 标记
│   └── 停止操作，提示: "请先运行 /cc-investigation:cold-start-interview 完成设置"
└── 配置就绪 → 继续
```

详细规则参见 `config-templates/config-loader.md`。

此技能当前无需读取特定配置项，仅检查配置就绪状态。

## When to Activate

- 设计具体调查执行方案时选择调查手段
- 需要跨领域调查工具知识时
- 评估调查资源（需要什么技术/工具/外部资源）
- 选择数字取证方案时
- 设计数据采集策略时

## 关键技术详述

### 财务数据分析

| 技术 | 适用场景 | 关键指标 |
|------|---------|---------|
| Benford定律 | 大规模数据异常检测 | 首位数字分布偏离 |
| JE测试 | 期末异常分录 | 非工作时间/非常用科目/非常用用户 |
| 比率分析 | 财务报表异常 | 毛利率突变/周转率偏离 |
| 资金追踪 | 资金流向 | 闭环/循环/异常通道 |
| T&E分析 | 费用报销 | 重复提交/超额/周末消费 |

### 数字取证流程

```
识别 → 保全 → 采集 → 分析 → 报告
```
- **识别：** 确定可能的证据源（设备/系统/云）
- **保全：** 写保护接入 → 计算哈希 → 镜像复制
- **采集：** 按照优先级收集（易失性数据优先：内存→网络连接→磁盘）
- **分析：** 文件系统分析 → 关键词搜索 → 时间线重构 → 关联分析
- **报告：** 取证过程记录 → 发现汇总 → 结论 → 保管链完整性声明

```

```


## 分析辅助工具

以下工具类型可辅助加速调查操作。**这些工具不是必须的**——未配置时，由模型按调查标准直接完成。

如环境中配置了以下类型的 MCP 服务器，可辅助加速：
- **网络搜索类 MCP**：可用于 OSINT 公开信息检索、背景调查
- **深度搜索类 MCP**：可用于更深入的互联网数据挖掘（通常需要 API Key）
- **邮件分析类 MCP**：可用于解析邮件头、分析元数据
- **图表生成类 MCP**：可用于生成调查发现的可视化展示

**工作流示意（OSINT 背景调查，工具可选）：**
1. 收到调查目标名称 → 搜索公开信息（由搜索类 MCP 辅助或浏览器手动搜索）
2. 获取搜索结果 → 读取具体页面内容
3. 如需更深层数据 → 尝试深度搜索（或进入公开数据库查询）
4. 汇总分析 → 输出背景调查报告

> 注意：MCP 可用性取决于环境配置。搜索工具不可用时，描述信息来源为"手动搜索"即可。
