# 渠道舞弊调查

适用于硬件（含服务）特价订单渠道销售场景的舞弊调查。

渠道销售舞弊的本质：**销售和（或）代理商利用特价申报环节与实际执行环节的信息不对称牟利——在申报端通过夸大竞争、虚构第三方成本、隐瞒渠道链路、强调终端客户价值来逃避合规审查、骗取更低折扣；在执行端偏离申报内容获取不当利益。**

调查的切入口：**查清特价项目是如何申报的，又是如何执行的，找到两者之间的偏差。**

渠道舞弊涉及六种常见模式，它们在同一个案件中往往同时出现（如虚报终端客户可能伴随渠道窜货，第三方成本造假可能伴随隐瞒渠道链路）。调查的关键是从单一信号出发，识别出可能共存的模式组合。

## When to Activate

- 接到特价订单相关的举报或线索
- 订单稽核发现异常交易模式（设备激活地址不匹配、渠道成本结构异常等）
- 渠道稽核发现窜货迹象或渠道链路异常
- 销售数据出现无法解释的异常（高返点、异常折扣、批量退货等）
- 需要设计渠道舞弊调查方案
- 需要调查渠道窜货、假单、虚假终端客户等特定问题

***

## 六种模式的框架定位

特价订单生命周期：**发起 → 申报 → 审批 → 执行**。

| 缺口       | 涉及模式                    |
| -------- | ----------------------- |
| **申报虚构** | 虚报终端客户、成本造假、拼单绑单、隐瞒渠道链路 |
| **执行偏差** | 渠道窜货、隐瞒渠道利润             |

***

## 六种舞弊模式

### 1. 虚报终端客户 (Fake End Customer)

**模式描述**：代理商或销售将产品申报给 A 客户，但货物实际交付给 B 客户（或根本无真实终端客户），以获取特价政策、返点或区域独占等不正当收益。

**关键信号：**

- 售后激活地址与申报 EndCustomer 名称/地址不符
- 收货单签收方与合同 EndCustomer 不符
- EndCustomer 的规模体量、业务场景、区位属性与报单需求显著不匹配
- 同一终端客户沿用低价，以扩容名义高频、大量采购

**调查切入点：**

- 比对售后系统设备激活记录与合同申报终端客户
- 核验终端客户工商注册信息（成立时间、注册资本、经营范围）
- 追踪合同交易的链路、货物流转和资金流转情况
- 安排产品巡检服务、访查终端客户是否真实使用该批设备

### 2. 渠道窜货 (Channel Diversion)

**模式描述**：代理商将限定区域/渠道的特价产品跨区域销售，破坏渠道价格体系和区域管理秩序。

**关键信号：**

- 设备激活地址与销售区域/授权范围不符
- 代理商存在跨区域发货的历史记录
- 二手市场出现本该定向供货的产品
- 不同区域出现相同序列号段的产品激活
- 非授权区域出现大量该品牌设备激活

**调查切入点：**

- 提取并分析设备激活地址与销售授权区域的交叉比对
- 追踪产品序列号的全生命周期（出厂→发货→激活→售后）
- 检查跨区域物流单据和运输记录
- 分析销售返利异常（窜货往往伴随不正常的返利申请）

### 3. 成本造假 (Cost Falsification)

**模式描述**：在申请特价订单中虚增第三方成本（或第三方服务费用），以加深折扣，增加渠道利润空间。

**关键信号：**

- 虚增垫子、仓储、驻场人员、营销费用
- 第三方供应商与代理商/销售存在隐性关联
- 无法提供第三方服务的真实交付凭证
- 成本结构与项目复杂度明显不匹配

**调查切入点：**

- 分析渠道成本结构，识别异常偏高的 OtherCostBucket 及其构成
- 核验第三方供应商的工商信息和实控人
- 要求提供第三方服务交付凭证（验收单、工作报告、工时记录）
- 追踪 OtherCostBucket 对应的资金流向

### 4. 拼单绑单 (Fictitious Bundled Needs)

**模式描述**：在真实项目中捆绑虚构或不需要的 SKU，多余的设备被挪用、出售或用于其他目的。

**关键信号：**

- 订单 SKU 清单中部分设备在终端客户处无安装/激活记录
- 合同配置与系统订单/BOM 配置存在不可解释的差异
- 设备数量明显超出终端客户实际需求
- 配置中包含价格高但非必需的附加模块

**调查切入点：**

- 比对合同+BOM+订单三者的配置一致性
- 验证每项 SKU 在终端客户处的安装、激活和使用情况
- 核验 BOM 与实际发货清单的差异
- 追踪多余设备的去向

### 5. 隐瞒渠道链路 (Undisclosed Channel Layer)

**模式描述**：代理商在申报时隐瞒真实的渠道层级（如隐瞒中间层、关联公司、非认证代理等），以获取更高折扣或满足渠道门槛。

**关键信号：**

- 发票流转链显示的实际层级数量多于申报层级
- 代理商关联网络中发现未申报的中间实体
- 发货路径中存在非认证或未申报的中间方
- 实际交易对手与合同载明的交易对手不一致

**调查切入点：**

- 分析发票链全链路，比对申报层级与实际层级
- 绘制代理商关联网络图，识别隐性中间层
- 对比发货记录中的签收链与合同申报的渠道结构
- 访谈关键节点的实际交易参与方

### 6. 隐瞒渠道利润 (Undisclosed Channel Profit)

**模式描述**：代理商或销售隐瞒下游的真实成交价格，使公司基于错误信息制定价格和折扣政策。

**关键信号：**

- 销售申报的用户成交价不知情，或者显著低于实际的用户成交价
- 渠道实际利润率与申报利润率存在显著差异
- 发现未披露的关联方交易或利益输送
- 资金流向与业务实质不匹配
- 存在异常的渠道费用、返点、市场基金支出

**调查切入点：**

- 比对申报利润率与实际利润率的差异
- 追踪资金流向，识别利益输送路径
- 分析返点和市场基金使用的合理性
- 核验关联交易的商业实质和定价公允性

## 调查方案设计要点

### 信号交叉验证

渠道舞弊的六种模式常常共存。当发现一个信号时，应同步排查其他模式的信号：

| 初始信号          | 通常伴随的模式         |
| ------------- | --------------- |
| 激活地址不匹配       | 虚报终端客户 + 渠道窜货   |
| 渠道成本异常偏高      | 成本造假 + 隐瞒渠道链路   |
| 配置与订单不一致      | 拼单绑单 + 可能伴随成本造假 |
| 渠道层级异常        | 隐瞒渠道链路 + 隐瞒渠道利润 |
| 最终用户成交价与申报不一致 | 渠道层级异常 + 渠道窜货   |

### 数据来源分类

渠道调查涉及三类数据来源，每类有不同的获取时机和方式：

| 数据类别       | 内容                            | 获取方式                    |
| ---------- | ----------------------------- | ----------------------- |
| **系统内数据**  | ERP 订单、售后激活记录、BOM 配置、员工 HR 档案 | 系统查询或导出（可在静默期完成）        |
| **接触获取数据** | 代理商合同、第三方服务凭证、签收单、补充说明        | 需接触当事方，在 FIELDWORK 阶段获取 |
| **公开查询数据** | 工商注册信息、股权结构、关联企业              | 公开渠道查询（可在静默期完成）         |

### 调查原则

调查用到的系统内数据遵循 **双轨获取** 原则：

- **主路径（primary）**：通过系统接口或 SQL 脚本自动获取，结果可重复验证
- **备用路径（fallback）**：主路径不可用时，由调查员从系统手动提取，结果须人工核验
- **不可获取时**：标注数据缺口（evidence\_gap），评估对 finding confidence 的影响

详细的通用调查原则（静默边界、缺口归因、信源前缀）参见 [`skills/investigation-foundation/SKILL.md`](../investigation-foundation/SKILL.md)。

## 分析辅助工具

以下工具类型可辅助加速渠道数据分析。**它们不是必须的**——未配置时，由模型按上述方法直接完成分析。

### 数据库查询类 MCP（如配置）

- **能力：** 执行 SQL 查询、导出数据
- **辅助场景：** 运行渠道分析脚本处理系统数据，将结果直接转为结构化证据
- **不可用时：** 从 ERP/售后系统手动导出数据，用电子表格或文本编辑器分析

### 网络搜索类 MCP（如配置）

- **能力：** 公开信息检索
- **辅助场景：** 工商注册信息查询、终端客户背景核查、壳公司特征识别
- **不可用时：** 浏览器手动查询工商网站

### 推理验证类 MCP（如配置）

- **能力：** 分步逻辑推演、多路径比对
- **辅助场景：** 跨模式信号交叉验证、资金流向逻辑推演、多项目并行分析
- **不可用时：** 由模型自行完成推演，输出推演过程供审查
